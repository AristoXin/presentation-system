# 用户最新要求锁定表

## 用户原话摘要

- 生产流程改为：各副主编组织部门会议，根据用户提供文件、初始稿件、之前已经生成的稿件等生成交付文件初始备忘录。
- 生产工作平台根据初始备忘录生产关键页面。
- 关键页面交各部门副主编初审；不合格则打回生产工作平台修改，再由同一副主编复审，直至各副主编一致同意交总编初审。
- 总编不同意时打回生产平台修改，再经过副主编初审、全部一致和总编复审，直至总编同意。
- 总编同意后，生产平台正式生产；正式生产后副总编和总编审核流程同前。
- 生产平台中，各个部门必须均有成员参与，这是必须执行的规定。
- 每个部门下设1-2名员工；部门内部会议需要副主编与成员达成一致意见。
- 所有subagent必须根据职责起中文名称，包括部门内部员工；HTML代码编写员工示例名为“蔻豆码”。

## 禁止事项

- 不得由主Agent直接生成、修改、截图或归档产物。
- 不得只有副主编参会或副主编自签部门意见。
- 不得缺少部门员工、员工中文名称、部门内部一致意见。
- 不得跳过交付文件初始备忘录、关键页试生产、副主编初审/复审、总编初审。
- 不得把关键页初审或总编初审冒充正式成品实现后复核。

## 本次执行理解

本次只进行Skill治理和证据补全，不生成或修改HTML/PPT/PDF/截图包等交付产物。新版规则必须贯穿入口、流程、团队职责、QA、版本证据和元数据，后续正式生产必须按“部门会议 -> 初始备忘录 -> 关键页试生产 -> 副主编/总编初审 -> 正式生产 -> 副主编/副总编/总编/QA/用户验收”执行。
