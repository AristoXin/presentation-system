# V6.8 跨项目失败经验固化修订摘要

> 日期：2026-06-20  
> 类型：Presentation System reference治理  
> 范围：跨项目设计、审美、动效、QA、参考库和证据链规则

## 修订原因

V0.13只把上一版问题作为当前项目失败处理，没有把用户点名的明显问题回写到长期设计参考文档。用户明确指出这些问题不应只针对某个客户培训项目修复，而应成为所有后续项目都不能再出现的通用经验。

## 从此跨项目生效的硬红线

- 已点名旧问题必须逐项关闭；旧问题在新版仍存在时，新版自动进入`FAILED_VISUAL_FREEZE`。
- 光学失衡、头轻脚重、左右失衡、主视觉漂移必须退回Page Spec。
- 多步骤页第1点、答案或后续阶段提前露出，必须退回首态/后续态矩阵。
- 流程页合并不同语义阶段，必须拆分状态、子页或讲者备注承载。
- 动效可运行但生硬，不得写动效通过。
- 用户说某页丑且不是单点bug时，必须重写视觉母线和页面族谱。
- 新版必须逐项关闭旧问题；只列新版变化不算关闭。
- 用户点名失败必须更新长期reference；只写当前项目失败审计不算经验固化。
