# 用户最新要求锁定表

## 时间

2026-06-21

## 用户最新原话摘要

用户指出当前HTML演示存在大量排版问题，不能把HTML当成传统PPT处理；动效不能只是简单按左键弹出；UI、审美、动效和文案部门必须深度协作；必须核实实现后复检是否有各subagent参与。用户建议新增“工作平台”部门：平时不启用，等各部门副主编达成一致意见后，抽调各部门员工进入工作平台，完成任务交付。

## 最高优先级要求

1. 当前任务先进入Skill治理和失败复盘，不继续补丁式生成HTML/PPT/PDF产物。
2. 正式生产必须区分“生产前输入”和“实现后复检”；生产前subagent意见不能冒充实现后复核。
3. 新增工作平台作为临时交付层，必须晚于部门一致、副总编一致性审核、总编方案闸门和制作工单。
4. 工作平台必须抽调各部门员工Agent或执行专员，形成跨部门真实交付，不允许主Agent单人生产。
5. 工作平台不能自审、自签通过，完成后必须交回原责任副主编做实现后复核。
6. 缺少工作平台编制、交付记录或文件所有权映射时，不得生成、截图、validation、命名或归档任何产物。

## 覆盖旧要求

本要求覆盖此前允许“执行层生成真实产物”的宽泛说法。新版规则要求：制作工单必须先进入工作平台启用检查，只有`PROCESS_READY_WORK_PLATFORM`成立后才允许真实产物生产。

## 当前执行理解

本轮只修订Skill系统文件和治理证据。V0.15只能作为失败证据和流程缺口样本；未获得用户明确恢复生产授权前，不进入下一版全稿、Style Frame、截图验证或QA冻结。
