# V0.15失败审计

## 结论

V0.15不应被视为候选稿、冻结稿或可验收版本。它只能作为`FAILED_VISUAL_FREEZE`下的失败证据。

## 用户点名问题

| 问题 | 用户反馈 | 治理定性 |
| :--- | :--- | :--- |
| 排版大量存在问题 | 截图P7出现大面积无组织留白、卡片关系松散、HTML空间利用不足 | 版式/UI与审美部门失效，触发Page Spec重写和光学平衡阻断 |
| HTML被当成PPT | 用户明确要求记住这是HTML | 交互工程和演示形态判断不足；不能只复制PPT静态页面逻辑 |
| 动效只是按左键弹出 | 动效“丑的要死” | 动效设计未形成运动语法、节奏和讲述目的 |
| UI部门失效 | 用户质疑UI部门 | UI实现后复核缺失，不能用工程烟测替代 |
| 复检subagent是否参与 | 用户直接追问 | 实现后复核未闭环；此前只有生产前输入，不能算复检 |
| 需要工作平台 | 用户提出新部门机制 | 组织流程必须新增临时交付层和硬门禁 |

## 根因

1. 生产链路把“执行层生成真实产物”写得过宽，给了主Agent直接生产的空间。
2. 协作记录虽然有生产前意见，但实现后同批subagent基于真实文件、截图和validation复核未完成。
3. QA链路允许工程烟测先行，未先卡住工作平台和实现后复核。
4. 组织架构缺少临时交付层，导致“部门一致意见”到“真实产物生成”之间责任断层。

## 本次治理动作

1. 新增工作平台硬门禁：未启用平台不得生产。
2. 新增工作平台编制表、交付记录、文件所有权映射。
3. 修改生产流程图：制作工单先进入工作平台，再生成真实产物。
4. 修改QA第0步：先检查subagent和工作平台证据，再允许截图、Playwright或validation。
5. 修改最终口径：缺少工作平台只能写`PROCESS_BLOCKED_NO_WORK_PLATFORM`。

## 恢复生产条件

恢复生产必须由用户明确批准。批准前只能提交失败审计、Page Spec、流程修订或受控的文字治理证据；不得继续生成下一版全稿。
