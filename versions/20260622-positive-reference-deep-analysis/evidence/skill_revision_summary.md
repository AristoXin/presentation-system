# 2026-06-22 Positive Reference Deep Analysis Revision

## 用户最新要求锁定

用户指出：碰到正向参考图时，不能只觉得好看或复用表层风格；必须思考为什么这样配色、文字如何配合、字体/大小/样式如何成立、版面如何规划、页面之间如何切换、如果由当前团队设计会考虑什么方案、如何设计动效，并进行详细拆解和经验总结。

## 规则结论

presentation-system现在把正向参考图视为设计决策样本，而不是视觉皮肤。任何正向参考图进入Style Frame、Page Spec或正式生产前，必须产出《正向参考图深拆与经验总结表》。

## 新增必填证据

- 配色逻辑：主色、辅助色、强调色、背景色和风险色的语义与比例。
- 文字协同：标题、正文、注释、标签、口播文案和按钮如何配合画面。
- 字体系统：字体气质、字号、字重、字距、行距和样式为什么适合当前受众和观看距离。
- 版面规划：视觉重心、网格、留白、图文比例、组件角色和阅读路径。
- 页面切换：跨页主题、色带、标题容器、图像槽和节奏变化如何延续或换气。
- 设计假设：如果当前团队重新设计，会考虑哪2-3个方案及其取舍。
- 动效方案：哪些内容静止、哪些分步、方向/时长/缓动/停顿/完成态如何服务讲述。
- 交互与HTML：若产物是HTML，必须说明输入、焦点、aria、回退、重播和export稳定态。
- 迁移结论：哪些经验可迁移，哪些表层元素必须舍弃，哪些需要品牌色、真实素材或内容关系改写。

## 验收口径

若正向参考只被总结为“好看、配色鲜明、版式活泼、有设计感”，没有回答配色、文字、字体、版面、跨页、替代方案、动效、交互和迁移边界，必须标记为“已观察，未吸收”，不得进入Style Frame、Page Spec、正式生产或QA冻结。
