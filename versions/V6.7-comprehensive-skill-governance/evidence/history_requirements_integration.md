# History Requirements Integration

Version: V6.7-comprehensive-skill-governance

## Integrated Historical Requirements

| Requirement | Governance Action |
| :--- | :--- |
| Do not keep patching failed artifacts before fixing the skill. | Skill governance mode blocks HTML/PPT/PDF/screenshot/sample production. |
| Real subagent participation and review must not be replaced by role-play. | Subagent Evidence Lock split into `SEL-real` and `SEL-downgraded`; downgraded output cannot freeze or formally deliver. |
| User-visible quality failures cannot receive conditional pass. | Optical balance, reveal state, motion beauty, and content copy failures are hard blockers. |
| Old versions, templates, and samples are only references. | Page count sufficiency, template suitability, and template transformation must be judged before production. |
| Style Frame after failure must not become a stealth full production run. | Recovery is split into failure audit, user-approved Style Frame exploration, and user-approved full production. |
| QA must judge design, not only run scripts. | QA now requires screenshots, contact sheet, manual focus-page review, optical balance evidence, reveal-state evidence, and motion beauty review. |
| Trends and visual references must be adapted, not copied. | Formal visual production requires trend suitability judgment; aesthetic failure or trend requests require main aesthetic trend research. |
| The system must decide if page count is sufficient. | Page count sufficiency is now a cross-department gate before page genealogy, Page Spec, Style Frame, and design engine selection. |

## Scope Confirmation

This governance update changed only the skill system and governance evidence. It did not create or revise deck artifacts.
