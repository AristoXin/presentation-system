# 历史对话需求整合表

版本：V6.5-visual-motion-failure-governance  
用途：Skill治理前置证据

## 已沉淀的长期要求

| 类别 | 历史要求 | 本轮整合 |
| :--- | :--- | :--- |
| 失败后停产 | 用户退回审美、流程或Skill可信度后，不得继续生成下一版全稿 | V0.12进入`FAILED_VISUAL_FREEZE`，本轮不创建V0.13 |
| 多Agent真实执行 | 不得由主Agent自审，不得用RACI表冒充副主编复核 | 新增部门签署失效规则，空泛条件通过无效 |
| 旧版否决 | 旧版只能作为失败证据，不得默认继承为母版 | V0.12标记为`FROZEN_FAILED_EVIDENCE` |
| 审美反AI | 不得只做卡片堆叠、模板复用、局部CSS微调 | 点名丑页必须退回视觉母线或Page Spec |
| 版式质量 | 不只检查对齐和溢出，还要检查页面重心和观看路径 | 新增光学平衡门禁 |
| 动效设计 | 动效必须服务讲述节奏，不得最后加统一动画 | 新增动效美感门禁和首态/后续态矩阵 |
| 分步呈现 | 多步骤内容必须由用户触发，不能提前露出答案或后续阶段 | 第5页、第9页问题写入Reveal硬失败 |
| 流行调研 | 当用户质疑主流审美或趋势调研时，不能只复用历史样张 | 后续恢复生产前必须完成主流审美趋势调研卡；外部搜索不可用时写阻断 |
| QA边界 | 自动验证只能叫工程烟测，不能替代审美QA | QA新增点名页、光学平衡、分步状态和动效美感复核 |

## 本轮新增高优先级要求

1. 第1页、第9页的排版失衡必须成为版式/UI和审美部门硬失败样例。
2. 第5页、第9页的分步缺失必须成为动效和交互工程硬失败样例。
3. 第15页“丑”必须作为Page Spec重写触发器，而不是局部修饰项。
4. UI、内容文案、审美、流行调研、动效部门必须提交可审计的页级证据。

## 冲突处理

若旧Skill规则允许“工程烟测通过但未冻结”的状态继续对用户展示，本轮修订后必须降级解释：该状态不能抵消用户审美退回；一旦用户退回，当前版本只能进入失败治理。
