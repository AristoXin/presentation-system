# 用户最新要求锁定表

版本：V6.6-page-sufficiency-governance  
状态：Skill治理，不恢复生产

## 用户原话摘要

- 用户一直要求进行判断，而不是限定于用户之前已经提供的版本或样本。
- 本轮重点是“页数是否足够”的问题。
- 页数充分性是跨部门协作问题，至少包括内容、审美、文案、UI排版等部门。
- 用户希望检查Skill内容，并对这部分进行更新。

## 禁止事项

- 不创建或修改HTML/PPT/PDF/截图包。
- 不按旧版、样本、模板或历史产物页数作为默认最终页数。
- 不把页数判断只交给需求部门或模板解析部门。
- 不用压缩字号、增加卡片密度、缩短文案来掩盖页数不足。

## 执行理解

本轮只修Skill治理规则。新增规则必须要求系统主动判断“页数是否足够”，并让内容、文案、审美、UI、动效、交互/交付和QA共同参与增页、拆页、合页、删页、附录或讲者备注承载判断。
