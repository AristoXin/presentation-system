# 页数充分性规则缺口审计

## 审计结论

原Skill已有“模板页数不是最终页数”和“页数预算”概念，但主要绑定在模板、旧PPT、参考deck、客户母版等输入场景下。缺口在于：当没有明确模板，或用户提供了旧版/样本作为参考时，系统仍可能默认沿用既有页数，而不是主动判断当前内容、文案、审美、UI和动效是否需要调整页数。

## 主要缺口

| 缺口 | 风险 | 修订动作 |
| :--- | :--- | :--- |
| 页数预算只在模板场景强制 | 非模板项目也可能内容超载或页数冗余 | 新增通用《页数充分性判断表》 |
| 旧版/样本页数容易被当成默认页数 | 继续被旧结构锁死，无法大刀阔斧重构 | 新增“不得默认沿用旧版/样本/模板页数” |
| 页数判断由需求或模板解析承担过多 | 内容、文案、审美、UI、动效的真实需求未进入判断 | 扩展为跨部门共同判断 |
| 页数不足被UI强行消化 | 小字、拥挤、卡片泛滥、动效步骤合并 | 新增页数不足阻断 |
| 证据链缺少页数充分性字段 | QA和版本归档无法追责“为什么是这些页” | 新增页数充分性证据规则 |
