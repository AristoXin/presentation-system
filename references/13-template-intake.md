# 13-template-intake

读取时机：当用户提供模板、旧PPT、参考deck、客户母版、品牌模板、示例演示文稿、固定页数要求或“参考这个风格/结构”时读取。

## 核心规则

- 模板页数不是最终页数。
- 模板风格不是必须照抄。
- 模板结构不是必须沿用。
- 先判断模板用途，再决定保留、提炼、改写、拆分或放弃。
- 视觉参考型模板必须进入参考经验转译：提炼结构、组件和节奏，结合当前用户资料和品牌色重做，不复制表层装饰。

## 模板用途分类

| 类型 | 判断信号 | 处理方式 |
| :--- | :--- | :--- |
| 品牌母版 | 公司VI、客户指定、固定Logo/页眉页脚 | 保留版心、Logo、安全区、页眉页脚和品牌规范 |
| 内容参考 | 模板中有可复用文案、图表、案例 | 执行保留、改写、删除、合并判断 |
| 结构参考 | 用户喜欢章节顺序或叙事骨架 | 参考结构，但按当前目标增删页面 |
| 视觉参考 | 用户喜欢风格、配色、版式 | 提炼可复用经验，输出参考经验转译卡，不机械套版 |
| 历史版本 | 旧稿、上一版、客户反馈版 | 保留有效资产，重建叙事和质量标准 |
| 客户强制模板 | 投标、客户规范、正式母版 | 严格保留强制项，非强制项按内容重构 |

## 操作步骤

1. 记录模板来源、页数、比例、格式和强制程度。
2. 建立页面族谱：封面、目录、章节页、内容页、表格页、案例页、结尾页。
3. 标注可复用页面、不适用页面、必须保留元素和可调整元素。
4. 若模板承担视觉参考作用，写清楚可借鉴经验、不可照搬元素、当前项目改写方式和品牌色取舍。
5. 判断最终页数是否应压缩、扩展、合并或拆分。
6. 输出模板解析表、页数预算表，并同步填写通用《页数充分性判断表》。
7. 若最终页数与模板页数差异明显，写明原因。

## 页数判断规则

- 用户给20页模板但内容只需要8页：说明压缩理由，如重复页面、内容不足、叙事链更短、可合并页面。
- 用户给5页模板但叙事需要12页：说明扩展理由，如证据链不足、关键决策页缺失、需要附录或案例页。
- 强制页数要求优先满足；超出信息进入附录、讲者备注或证据包。
- 模板页数预算只回答“模板与当前内容是否匹配”；通用《页数充分性判断表》还必须回答内容、文案、审美、UI、动效和QA是否认为当前页数足够。两者缺一不可。
