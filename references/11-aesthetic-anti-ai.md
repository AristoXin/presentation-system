# 11-aesthetic-anti-ai

读取时机：当用户要求“更高级”“不像AI做的”“提升审美”“避免模板感”，或任务涉及完整视觉生产、Style Frame、品牌气质、视觉方向确认、真实素材审查时读取。若涉及design-dna、PPT-DNA、guizang、HTML/PPT/Slides视觉生产或用户质疑审美/UI/动效没有深度集成，同时读取`20-design-dna-guizang-integration.md`。若用户指出排版/UI不好看、动效单一，或质疑stylekit/taste/frontend-slides/html-ppt/huashu-design等Skill没有集成，同时读取`21-design-skill-stack-integration.md`。

## 设计目的

演示文稿不得只做到“能看、能翻页、信息完整”。正式交付必须避免明显的AI生成感、模板感和组件堆叠感。任何项目进入完整页面生产前，必须先通过审美策略与反AI味评审。

本机制重点解决：

- 页面看起来像通用AI模板。
- 过度依赖渐变、圆角卡片、阴影、胶囊标签和伪图表。
- 品牌识别弱，只是把公司名放进Logo占位框。
- 页面结构重复，连续多页都是卡片网格。
- 案例页、模型页、行动页缺少真实叙事张力。
- UI看起来很满，但没有高级感和可信度。
- HTML演示控件默认强显示，破坏页面沉浸感和截图干净度。
- 页面排版虽然对齐但光学失衡，例如头轻脚重、头重脚轻、左重右轻、右重左轻或主视觉漂移。
- 动效虽然可运行但生硬，页间和同页切换缺少运动语法、节奏和讲述美感。
- 多步骤页面默认露出应由用户触发的第一点、答案或后续阶段。
- 把参考图当成可复制模板，导致页面只像参考、不像当前项目。
- 团队名义上调用了审美/UI/QA部门，实质仍在维护旧模板。
- 旧版本被当作默认母版，缺少推翻旧方向的勇气和机制。
- 自动验证通过，但真实观看时色彩割裂、排版压迫或视觉不舒服。
- 用户指出的明显审美/版式/动效问题只停留在当前项目失败审计，没有进入跨项目设计经验，导致其他项目复现同类错误。
- StyleKit只被当作提示词或风格slug记录，没有转译为审美意图、token层级、组件语法、状态/动效约束和QA测试。
- PPT-DNA只输出抽象JSON或形容词，没有变成当前项目的token、页面族谱、组件规范和效果降级策略。
- guizang只被当作瑞士风/杂志风模板套用，没有根据用户资料完成A/B适配、主题节奏、逐页layout/Sxx映射和模板改造差异。
- UI部门把设计理解成方块、配色和文字填充，没有跨页视觉语法、信息层级、真实视觉锚点、动效/静止策略和多部门协同输入。

## 反AI味负面清单

以下视觉模式不得默认使用。若确需使用，必须由品牌与审美策略副主编说明理由并经总编确认。

| 高风险模式 | 问题 |
| :--- | :--- |
| 大面积蓝紫渐变背景 | 容易产生通用SaaS模板感 |
| 玻璃拟态面板 | 容易显得过时或AI默认风 |
| 通用网格背景 | 容易像模板库作品，缺少项目绑定 |
| 随机装饰线 | 装饰多于信息，削弱专业可信度 |
| 无意义斜切色块 | 看似有设计，实际不承载信息 |
| 全页圆角卡片堆叠 | 页面像组件库拼装，缺少叙事结构 |
| 每页都是“标题 + 卡片 + 图标” | 结构重复，无法服务不同信息任务 |
| 胶囊标签过多 | 信息层级碎片化，缺少正式感 |
| 多色彩虹模型 | 看似丰富，实际削弱品牌和语义 |
| 圆环/五边形万能模型 | 模型关系表达不准确，模板感强 |
| 星级评分 | 容易显得游戏化、缺少专业判断 |
| 假线条占位文档 | 降低案例真实性和可信度 |
| 重阴影和发光效果 | 造成廉价科技感 |
| 每页同一种卡片网格 | 缺少节奏变化和设计定制感 |
| 常驻底部控制栏 | 破坏正式演示沉浸感 |
| Logo文字框占位 | 缺少真实品牌系统 |
| 通用科技感/商务感/咨询风 | 用风格标签替代项目判断 |
| 低对比文字 | 黑底黑字、深底深字、浅底浅字会直接破坏可读性 |
| 无语义多色线条 | 黑、红、绿等线条混用会显得混乱且廉价 |
| 弱表头表格 | 表头不显眼、正文对齐混乱会让页面像未设计的表格粘贴 |
| 大面积黑/白/高饱和蓝硬切换 | 造成阅读疲劳和割裂感，尤其在演示翻页时不舒服 |
| 单页内黑白蓝等强反差色块并列 | 看起来像多个模板拼接，缺少统一视觉母线 |
| 为了显得“瑞士/科技/高级”使用强蓝 | 容易让accent变成背景噪音，削弱内容可读性 |
| 方块配色填字式UI | 只完成摆放，没有页面角色、信息植入、视觉锚点和动效协同 |

## 正向设计标准

1. 页面必须有清晰的信息层级：主标题、主观点、证据、辅助说明、页脚。
2. 每页必须有一个明确视觉重心，而不是多个模块平均分布。
3. 留白必须稳定，版心必须统一。
4. 色彩必须克制，优先使用1个主色、1个辅助色、1个风险/强调色。
5. 卡片、表格、标签、进度条等组件必须统一，不得像不同组件库拼接。
6. 不同页面应有不同叙事结构，而不是复制同一版式。
7. 视觉设计必须服务当前用途，客户汇报、培训、路演、研究报告、复盘不能共用同一种模板。
8. 设计应有人工设计痕迹：克制、取舍、对齐、节奏、层级、细节，而不是生成式堆叠。
9. 参考图只能被转译为经验：结构、节奏、组件语法、色彩策略和风险边界；不得照搬参考图的品牌、行业语汇、装饰、图标、文案或无关配色。
10. 当前用户资料必须成为视觉锚点来源：真实品牌色、真实数据、真实截图、真实案例、真实受众和真实交付场景优先于任何参考风格。
11. 可读性先于风格：任何文本、表格和页脚必须与底色形成清晰对比，不得为了视觉氛围牺牲阅读。
12. 线条系统必须克制统一：普通分隔线用中性色或品牌辅助色，红/绿等强语义色只用于明确状态。
13. 表格必须像被设计过：表头层级清楚，正文对齐按字段类型统一，重点列和注释有一致规则。
14. 视觉舒适度必须先于风格标签：同一deck应有稳定的明度节奏，避免连续大面积纯黑、纯白、高饱和蓝互相硬切；重点页可以有强对比，但必须有缓冲页、留白和统一色温。
15. 配色必须可解释：主色、辅助色、强调色分别对应品牌、结构或状态；若只是为了“高级/科技/瑞士”而加入黑、白、蓝等冲突色块，审美副主编必须否决。
16. 排版必须有呼吸感：中文正文不得被窄栏、等分柱、拥挤卡片或连续粗体压迫；投屏观看时，3秒内应能分辨主判断、证据和行动。
17. 排版必须有光学平衡：页面的视觉重量、留白、主视觉和文字块应形成稳定张力。视觉重心漂移、单侧过载、底部压重或顶部空洞，即使对齐正确，也属于审美失败。
18. 动效必须有审美质量：转场和分步应形成现场讲述的节奏，不得只靠统一淡入、上移、硬切或默认transition制造“有动画”的假象。
19. 分步呈现必须尊重观看 suspense：观众需要主动触发的第一点、判断、答案和后续阶段，初始态不得完整显示。
20. 正向参考必须被拆成跨页系统：先拆页面角色，再提炼网格、标题容器、视觉锚点、组件语义、动效触发和交付路线。只学习参考图的颜色、贴纸、斜切色块或表层热闹感，视为未吸收。
21. 正向参考必须被拆成设计决策：为什么这样配色、文字如何配合、字体/字号/字重/样式为什么成立、版面如何规划、页面之间如何切换、如果当前团队来做会考虑哪些方案、动效与交互如何设计。缺少这些追问时，不得称为“已学习设计理念”。

## Style Frame闸门

完整HTML、PPTX、PDF或截图包生产前，必须按`00-production-gate.md`的任务等级产出或验证Style Frame：单页/小修至少1个方向，正式视觉生产和用户审美退回不少于2个方向。Style Frame不要求覆盖全部页面，但必须覆盖或说明：

- 字体系统；
- 颜色系统；
- 页眉页脚；
- 标题层级；
- 卡片/表格/引用/图表组件；
- 控件显示方式；
- 页面留白和版心；
- 是否存在AI化视觉风险。
- 是否已经完成参考经验转译：借鉴什么、不照搬什么、当前资料如何改写参考。
- 若采用正向参考，是否完成《正向参考图深拆与经验总结表》：配色、文字、字体、版面、跨页切换、替代方案、动效、交互和迁移边界。
- 是否完成旧版否决：旧版哪些方向不能继承，新版在哪些层面发生不可逆改变。
- 是否完成动效方向：哪些页面动、为什么动、哪些页面必须静止。
- 是否完成深度定制证明：页面族谱、视觉锚点、动效脚本和组件语法是否来自当前内容，而不是来自模板可用性。
- 是否完成模板适配性否决：哪些模板、历史样张或流行趋势被明确弃用，为什么不适合当前内容。
- 是否完成模板改造差异：即使采用模板，哪些结构被重写、新增或删除，为什么这些调整来自当前内容。
- 是否完成StyleKit审美组件转译：若涉及`stylekit-skill`或`REF-20260621-01`，必须记录意图、记忆钩子、token层级、组件架构、状态转译、quality_gate和swap/squint/signature/token测试。
- 是否完成PPT-DNA三维设计身份：若涉及`PPT-Design-DNA-main`、`design-dna`或`REF-20260621-03`，必须记录design_system、design_style、visual_effects如何进入页面族谱、UI方向、动效脚本和QA。
- 是否完成guizang深度版式转译：若涉及`guizang-ppt-skill`或`REF-20260621-02`，必须记录A/B适配理由、主题节奏、逐页layout/Sxx映射、图片槽位、动效意图和模板改造差异。
- 是否完成`20-design-dna-guizang-integration.md`规定的统一吸收合同：Presentation Design DNA档案、DNA来源追踪表、设计引擎桥接表、UI吸收表、动效吸收表和QA吸收检查。
- 是否完成`21-design-skill-stack-integration.md`规定的Design Skill Stack决策：StyleKit候选/quality_gate、TasteSkill三拨盘/pre-flight、演示引擎选择、Motion source map和Skill Stack QA吸收。

Style Frame必须由多部门共同生成，不得由视觉或UI单部门先出图、再让其他部门事后盖章。生成截图前必须填写以下协同输入：

| 部门 | 必须输入 |
| :--- | :--- |
| 语言内容 | 每个样张方向的一句话主判断、页面信息载荷、标题/注释/提问句、讲述动作 |
| 品牌审美 | 参考经验转译、趋势取舍、视觉母线、品牌色与禁用风格、StyleKit意图/记忆钩子/反泛化测试、PPT-DNA design_style、guizang A/B适配理由 |
| 版式/UI | 栅格、字阶、组件语法、控件状态、表格/卡片/标注规则、StyleKit token层级与Base→Variant→Size→State→Override约束、PPT-DNA design_system、guizang layout/Sxx映射 |
| 动效设计 | 分步显隐、转场节奏、静止页清单、export稳定态、StyleKit state/interaction演示转译、PPT-DNA visual_effects、guizang动效recipe/低功耗策略 |
| 交互工程 | HTML/PPT实现方式、投屏/截图/导出约束、guizang模板/校验/截图模式 |
| QA | 截图有效性、可读性、对比度、重点页人工检查项、StyleKit quality_gate与swap/squint/signature/token验证、PPT-DNA字段落地、guizang主题节奏/版式登记 |

Style Frame协同输入还必须包含《多页视觉系统拆解表》和必要的《正向参考图深拆与经验总结表》：若参考或目标产物包含多页PPT，语言内容先给页面任务和讲述动作，版式/UI再给跨页视觉语法，动效设计再给首态/后续态，交互工程再判断HTML/PPTX/图片PPT路线。若采用正向参考，品牌审美必须解释配色与字体，语言内容必须解释文字如何配合画面，版式/UI必须解释版面规划，动效和交互必须解释切换、状态和HTML行为。任何部门缺席时，不得称为“已经学习参考设计理念”。

若任一适用部门未输入，当前产物只能称为“视觉方向草图”，不得称为Style Frame。若部门意见冲突，先修改Page Spec或视觉策略；不得让总编直接拍板覆盖副主编反对意见。

### StyleKit审美组件应用

当项目需要Style Frame方向、风格候选、风格混合、设计库校准，或用户点名StyleKit/设计库/风格提示词时，`stylekit-skill`必须被当作审美组件来源处理，而不是生成提示词后直接落图。

| 部门 | 应用方式 | 不合格信号 |
| :--- | :--- | :--- |
| 品牌审美 | 从StyleKit提取purpose、audience、tone、memorable hook，并决定当前deck的视觉母线、禁用风格和反泛化测试 | 只引用slug、流行词或“高级感”，没有当前项目的记忆钩子 |
| 版式/UI | 把StyleKit的token hierarchy和component architecture转成颜色语义、字阶、间距、表格/卡片/标注/控件组件规则 | 组件只是换皮；variant、size、state、override没有边界 |
| 动效设计 | 把state/interaction转成首态、揭示态、强调态、静止页、export稳定态和reduced-motion策略 | 把网页hover/focus机械搬进演示，或分步状态无法导出稳定截图 |
| 交互工程 | 对HTML演示保留必要的focus-visible、键盘/触控状态和导出模式差异；对PPT/PDF说明不适用状态 | 交互状态与截图/投屏/导出互相冲突 |
| QA | 逐项核对quality_gate、swap/squint/signature/token测试、token所有权和状态转译证据 | 只有“已调用StyleKit”，产物看不出规则执行 |

StyleKit不得覆盖当前用户资料、品牌色、受众场景和内容关系。若StyleKit候选风格与当前资料冲突，品牌审美副主编必须弃用并记录原因；若采用，必须说明哪些规则被改写为页面族谱、UI方向和动效脚本。没有这些转译时，记录为“已读取未吸收”，不得进入审美通过。

### PPT-DNA与guizang深度设计应用

当用户提供资料、旧稿、参考图、品牌规范或要求“不要做丑/要真正有审美和排版设计”时，品牌审美和版式/UI部门必须优先判断是否需要PPT-DNA和guizang参与。它们的职责不是替代设计判断，而是迫使设计判断可追溯、可映射、可QA。

| 部门 | PPT-DNA应用 | guizang应用 | 不合格信号 |
| :--- | :--- | :--- | :--- |
| 品牌审美 | 提取design_style：mood、genre、visual language、composition、brand voice，并写成视觉母线 | 判断A电子杂志/B瑞士国际主义是否适合当前资料、受众、素材和讲述场景 | 只说“高级/瑞士/杂志感”，没有用户资料驱动的取舍 |
| 版式/UI | 提取design_system：颜色、字体、间距、layout、shape、components，并转成组件规范 | 建立页码→页面角色→内容关系→layout/Sxx→选用理由→图片槽位映射 | 没有逐页映射，或用同一卡片网格消化所有内容 |
| 动效设计 | 提取visual_effects和motion，判断效果强度、性能层级、fallback | 将guizang动效recipe转成讲述动作、首态/后续态、export稳定态和低功耗说明 | 统一fade-up、过度WebGL、导出中间态或无静态版本 |
| 交互工程 | 根据DNA effects决定HTML/PPT/PDF可实现边界 | 执行模板类名预检、瑞士风data-layout登记、校验脚本或等价检查 | 类名缺失、A/B混用、未登记版式、截图无法验证 |
| QA | 核对DNA字段是否落到产物，效果是否有降级 | 核对主题节奏、单一accent、字号/字重、图片比例、底部安全区、逐页截图 | 产物像模板套皮；流程记录无法解释为什么这样排版 |

若PPT-DNA或guizang参与了方向选择，却没有改变页面族谱、组件语法、主题节奏、动效脚本或QA字段，必须记录为“已提取未吸收/已选模板未吸收”，不得作为审美通过证据。

完整项目建议覆盖封面页、核心模型页、重点案例页、数据/表格页、收束行动页。小任务可以输出简化Style Frame或Page Spec，但必须经总编记录原因，且不能跳过审美判断。

若用户已退回“丑、难看、像模板、像资料页、没有审美”，恢复流程分三段执行：

| 阶段 | 允许产出 | 禁止动作 |
| :--- | :--- | :--- |
| 失败审计期 | 失败审计、旧版否决、文字Page Spec、版式草图说明 | 生成HTML/PPT/PDF/截图包、命名Vx、跑Playwright/validation |
| 用户批准Style Frame探索期 | 在`SEL-real`、制作工单、工作平台编制、文件所有权映射和`PROCESS_READY_WORK_PLATFORM`齐全后，不少于2个方向的受控局部视觉证据；可为1-3页HTML样张截图、PPTX截图、Figma截图或清晰版式草图 | 缺少subagent或工作平台证据时生成文件/截图；扩展全稿、命名候选交付、QA冻结、复制旧版作为母版 |
| 用户确认恢复全稿生产期 | 按正式生产流程生成全稿HTML/PPT/PDF和截图证据 | 跳过页数充分性、subagent复核、Style Frame协同输入 |

不得直接进入全稿，不得只用文字描述“瑞士风、杂志风、高级感”。

用户审美退回不是自动迭代许可。退回后不得直接生成下一版全稿；“下一版本的Style Frame”仅指候选视觉方向证据，不代表恢复HTML/PPT/PDF生产。只有当用户看过失败归档和候选方向，并明确批准“恢复全稿生产/进入下一版全稿”后，才可扩展为全稿。

若用户指出“上一版问题没解决”，不得用新的视觉命题、换色、换版式名称或局部组件增强替代问题关闭。必须先完成《上一版问题关闭矩阵》，逐项证明旧问题已被Page Spec、视觉母线、动效脚本、截图证据和同类页联动关闭；未关闭的项目必须标为`FAILED_VISUAL_FREEZE`，不能称为新版候选。

若用户退回的是“看着不舒服、黑白蓝混用、排版瞎搞”等视觉舒适问题，下一轮Style Frame必须先给出《视觉舒适度检查》：主背景、正文背景、强调色、文字色、明度节奏、观看距离、连续页面色温变化、强对比使用理由。没有该表，不得进入全稿。

Style Frame必须是真正的视觉证据，不只是文字描述。正常生产或用户批准Style Frame探索期，在真实subagent证据、制作工单、工作平台编制和文件所有权映射齐全后，可接受证据包括：1-3页HTML样张截图、PPTX截图、Figma截图或清晰的版式草图。失败审计期尚未获用户批准探索时，只能提交文字Page Spec或版式草图说明，不能生成新的HTML/PPT/PDF/截图包；缺少工作平台证据时也不能生成文件或截图。只写“稳健专业、克制高级、反AI味”这类形容词，不算通过。

若基于既有HTML版本迭代，Style Frame闸门必须先回看旧版关键页。检查顺序：

1. 先看封面是否有品牌识别和正式场景感。
2. 再看3页以上连续内容页是否版式重复。
3. 再看核心模型页是否只是彩色条目或万能模型。
4. 再看案例/系统页是否有真实素材，或只是“待替换”大占位。
5. 最后看收束页是否有行动层级，而不是又一组卡片。

任一项失败，先重做Style Frame，不进入全稿CSS微调。

每个项目至少提供两个视觉方向：

| 方向 | 说明 |
| :--- | :--- |
| 稳健专业型 | 适合客户汇报、内部汇报、研究报告 |
| 叙事培训型 | 适合授课培训、工作坊、能力提升课程 |
| 高冲击发布型 | 适合产品发布、路演、品牌宣讲 |

总编确认视觉方向前，不得进入全稿HTML或PPTX生产。

## 旧版否决门

当用户明确说“前版只是参考”“像模板”“没有大刀阔斧改变”“审美/UI部门只调用过去模板”时，必须先进入旧版否决门，不得继续微调旧版CSS。

旧版否决门必须回答：

| 字段 | 要求 |
| :--- | :--- |
| 旧版根本错误 | 不是列小bug，而是指出方向、结构、组件或流程的根问题 |
| 不可继承项 | 哪些颜色、组件、线条、动效、版式、文案节奏不能继续使用 |
| 可继承项 | 哪些内容骨架、交互骨架、证据逻辑可保留 |
| 新版不可逆改变 | 至少列出3个层面：视觉母线、页面族谱、组件语法、动效脚本、语言组件等 |
| 审美部门反对意见 | 审美副主编必须可退回旧方向，不得只写“条件通过” |
| UI部门反对意见 | UI副主编必须指出组件级问题，如表格对齐、术语组件、线条语义 |
| QA否决条件 | QA必须说明哪些问题即使脚本通过也不能冻结 |

旧版否决门未通过，不能进入Style Frame，更不能进入HTML创建。

## 经验固化门

审美退回、用户点名明显问题或多版本反复失败后，必须把失败经验固化到长期reference，而不是只写在当前项目记录里。

| 失败经验 | 必须固化为 | 适用范围 |
| :--- | :--- | :--- |
| 头轻脚重、左右失衡、主视觉漂移 | 光学平衡硬失败与三分区复核方法 | 所有封面、章节页、模型页、案例页、收束页 |
| 第1点/答案/后续阶段提前出现 | 首态/后续态矩阵和分步QA硬失败 | 所有流程、清单、模型、演练、AI复核页 |
| 页间或同页切换生硬 | 动效美感门禁和运动语法要求 | 所有HTML演示和动画型PPT |
| 某页“丑”但不是单点bug | 视觉母线和Page Spec重写要求 | 所有完整视觉生产 |
| 部门签署后仍有明显问题 | 条件通过失效与实现后复核要求 | 所有团队协作生产 |

经验固化必须至少更新一个核心reference，并在版本证据中记录更新路径。只在README、QA记录或失败审计里写“下次注意”，不算固化。

## 主流审美趋势调研卡

正式视觉生产至少必须完成《趋势适用性判定卡》，说明是否需要外部近期验证、哪些趋势适合、哪些趋势必须弃用。用户要求“当前/主流/流行/最新审美”、要求不要只复用过去模板、发生审美退回或模板质疑时，品牌审美副主编必须进一步完成本《主流审美趋势调研卡》，再进入Style Frame。

当用户质疑“有没有审美学习/有没有搜索趋势/有没有调用经验库”时，也必须完成本卡。外部搜索或近期参考验证可用时必须执行；不可用时必须写入阻断，不能用旧参考库或模型记忆冒充“趋势调研”。

| 字段 | 要求 |
| :--- | :--- |
| 调研来源 | 外部搜索、近期优秀案例、已沉淀参考库条目；若外部搜索工具不可用，必须按降级审批暂停 |
| 趋势摘要 | 只总结对当前任务有用的趋势，不堆砌流行词 |
| 适用边界 | 哪些趋势适合当前受众、内容密度、观看距离和品牌语气 |
| 明确不用 | 哪些流行做法会导致模板感、低可读性、过度装饰或与资料冲突 |
| 当前项目改写 | 说明趋势如何被改写为页面族谱、组件语法、色彩、字体、动效和信息密度 |
| 部门异议 | 语言内容、版式/UI、动效设计和QA可提出反对意见；反对意见未解决不得提交副总编 |

趋势调研卡不是“找几个好看案例”。它必须回答为什么当前项目需要这些趋势、为什么有些趋势必须舍弃，以及新版与旧版相比发生了哪些不可逆改变。

## 页面结构重构优先

如果页面不好看，不得优先通过微调颜色、阴影、边距解决。必须先判断是否存在结构性问题：

- 页面核心观点是否单一？
- 是否所有内容都被装进卡片？
- 是否存在真实视觉锚点？
- 标题、证据、结论、行动之间是否有清晰层级？
- 页面是否有留白节奏？
- 当前版型是否服务于内容，而不是套用了默认组件？

若结构不成立，必须重写Page Spec，再进入视觉实现。

## 演示类设计引擎门禁

涉及HTML/PPT/Slides视觉生产时，品牌审美副主编必须选择一个主设计引擎，并把引擎规则转译成当前项目的页面族谱。可选主引擎包括：`guizang-ppt-skill`、`html-ppt`、`frontend-slides`、客户母版或项目自有设计系统。未选择主引擎，不得直接手写全稿样式。

选择主设计引擎必须后置于部门深度定制。品牌审美副主编不得先宣布“采用某风格/某主题色/某模板”，再要求内容、UI、动效部门配合填充。正确顺序是：当前材料的内容关系和使用场景先生成页面族谱；参考库和趋势调研再给出可借鉴与不可照搬项；UI和动效部门定义组件语法与讲述节奏；最后才选择设计引擎作为实现边界。任何把设计引擎当设计答案的记录，必须退回。

参考库与流行调研的职责不是寻找可套用模板，而是建立适配和否决机制。审美副主编必须说明哪些参考经验适合当前项目、哪些不适合、为什么不适合；流行趋势必须被翻译成当前内容的结构规则，而不是被复制成视觉表层。若一个趋势或模板会削弱证据可信度、训练动作、讲述节奏、品牌语气或投屏可读性，必须否决。

即使某个模板、风格或Sxx版式大方向合适，也必须进行审美改造。审美副主编要判断它的默认比例、信息密度、留白、组件语法、视觉锚点和动效是否需要为当前内容调整；UI副主编要给出改造差异；动效副主编要说明默认动效是否保留、减弱、替换或删除。未经改造的模板不得进入Style Frame。

### 使用 guizang-ppt-skill 的硬规则

| 项目 | 要求 |
| :--- | :--- |
| 风格选择 | 必须明确选择风格A电子杂志或风格B瑞士国际主义；一份deck不能混用两套表层语言。 |
| 瑞士风 | 必须有版式登记或映射表，例如页码 → Sxx / 页面角色 / 为什么选它。不得自造“五列卡片”“左右档案表”等未登记结构后宣称是瑞士风。 |
| 深度定制 | 必须先有页码 → 当前内容关系 → 页面角色 → 视觉锚点 → 动效意图，再映射到Sxx。不得先选Sxx、IKB或瑞士风，再把用户内容塞进去。 |
| 适配性否决 | 每个采用的Sxx必须说明为什么适合当前内容；每个明显诱人的但不适合的Sxx也要写弃用理由。不得因为瑞士风版式存在就硬套。 |
| 改造差异 | 每个采用的Sxx必须说明保留什么、改写什么、新增什么、删除什么，以及这些改造如何服务当前内容。不得原样调用。 |
| 主题节奏 | 必须先列主题节奏表：hero / light / dark / 工作台 / 案例 / 收束，避免连续浅底资料页。 |
| 色彩 | 一份deck只保留一个accent；不得红/绿/金/蓝多色混用。 |
| 字体 | 瑞士风全程无衬线，大标题轻字重；不得用衬线粗黑标题冒充高级感。 |
| 动效 | 每页动效必须服务讲述动作：提问、等待、揭示、标注、对照、收束；不得只有统一fade/rise。 |
| 验证 | 必须运行或等价执行guizang瑞士风校验：版式映射、单一accent、截图非空、重点页人工复核。 |

### 失败样式黑名单

以下模式一旦出现在关键页，必须退回Page Spec而不是调CSS：

- 五等分窄柱承载中文标题和段落。
- 左右两栏同权表格承载案例材料和判断结论。
- 连续3页以上浅底、标题、卡片/表格结构。
- 每页多个黑底/深色大块争夺视觉主权。
- 大标题、模块标题、正文全部加粗，缺少字重层级。
- “训练页”没有可填写动作、示范答案、讲师引导和学员产出。
- 截图证据为空白或过渡帧，但QA仍写通过。
- 黑色、白色、高饱和蓝在同一页或相邻关键页无语义地大面积混用。
- 只通过换主题色、改CSS变量、调间距来回应审美退回，没有重写页面族谱和视觉母线。

## 视觉舒适度检查

审美副主编在Style Frame和实现后复核都必须填写：

| 字段 | 要求 |
| :--- | :--- |
| 明度节奏 | 说明深色、浅色、强调页如何分布，是否存在连续硬切 |
| 色彩角色 | 主色/辅助色/accent/风险色分别承担什么语义 |
| 禁用色组合 | 明确哪些黑白蓝、红绿、冷暖冲突组合不得出现 |
| 中文阅读舒适度 | 正文栏宽、行距、字重、留白是否适合投屏和快速浏览 |
| 页面呼吸感 | 是否存在拥挤卡片、窄栏正文、连续粗体和多重边框 |
| 截图证据 | 至少封面、核心模型页、案例页、工作台/系统页、收束页 |
| 结论 | 通过/退回视觉母线/退回Page Spec/阻断 |

该检查不能由主Agent代签。若结论不是“通过”，不得进入全稿或冻结。

### v3式失败信号

以下信号来自HTML演示迭代中常见失败模式。出现任一项，都按结构问题处理：

- “功能都齐了”，但QA只验证按钮、页数和脚本，没有审美截图证据。
- 正式关键页仍出现“Logo待替换”“截图待接入”“真实简历截图待替换”等大面积占位，却被标为通过。
- 多页反复使用浅底卡片、深色侧栏、彩色上边框、数字方块，页面换标题但视觉关系不变。
- 全文大量使用超粗字重和大字号，造成每个模块都在喊，反而没有主次。
- 背景网格、斜切色块或纹理在多数页面出现，但与业务信息没有对应关系。
- 颜色数量看似克制，实际每个组件都用同一套色条，形成“咨询模板感”。

## 真实素材优先

关键页面不得依赖伪造素材或抽象占位。

| 页面类型 | 禁用做法 | 推荐做法 |
| :--- | :--- | :--- |
| 简历案例页 | 灰线模拟简历 | 脱敏真实简历截图或结构化证据卡 |
| 数据页 | 随机图表占位 | 使用真实数据或明确标注样例数据 |
| 品牌页 | 文字Logo框 | 使用真实Logo、VI色和品牌图形系统 |
| 二维码页 | QR占位框 | 若无二维码，应改为“会后材料入口说明” |
| 案例复盘页 | 大框+口号 | 信号 → 判断断点 → 后果 → 复盘原则 |

## 审美验收标准

品牌与审美策略副主编必须从以下维度验收：

- 是否一眼看出品牌气质；
- 是否避免通用AI模板感；
- 是否存在过度通用的渐变、玻璃拟态、网格背景、随机装饰线；
- 是否存在无意义斜切色块；
- 是否每页都是“标题 + 卡片 + 图标”的模板结构；
- 是否颜色过多、阴影过重、留白不稳；
- 是否装饰多于信息；
- 是否标题很大但信息组织薄弱；
- 是否图标和色块没有业务含义；
- 是否缺少与具体项目材料绑定的视觉证据；
- 是否像模板库作品，而不是为当前受众和场景定制；
- 是否模块之间视觉权重平均，缺少主视觉、主结论和阅读路径；
- 是否有明确页面节奏变化；
- 是否色彩克制且有语义；
- 是否字体、留白、网格稳定；
- 是否存在黑底黑字、深底深字、浅底浅字或图片上文字不可读；
- 是否存在无语义多色线条、边框厚度混乱或黑/红/绿混用；
- 是否存在表格表头不显眼、正文对齐规则混乱；
- 是否每页都有真实信息承载；
- 是否控件和动效服务演示而非炫技；
- 是否适合当前场景，而不是套用通用网页风格。

未通过审美验收的页面，不得进入QA终审。

审美验收必须输出“退回清单”，不能只写通过/不通过。每个退回项必须落到页面或组件族：

| 退回类型 | 触发信号 | 修复方向 |
| :--- | :--- | :--- |
| Style Frame退回 | 整体气质像模板、品牌不明显、颜色/字体不成立 | 重做视觉方向，不改局部CSS |
| Page Spec退回 | 内容关系不清、页面无主视觉、卡片矩阵承重 | 重写页面角色、核心观点和版式策略 |
| Typography退回 | 标题过大、正文过粗、行长过长、层级混乱 | 改文案、降字号、建立字阶和行距 |
| Contrast退回 | 黑底黑字、深底深字、浅底浅字、图片上文字不可读 | 重设底色/文字色/遮罩，保证投屏可读 |
| Line System退回 | 同页或同组件族无语义混用黑/红/绿线条，边框粗细混乱 | 建立线条色阶和语义，只保留必要分隔 |
| Table退回 | 表头不显眼、正文对齐混乱、数字文本规则混用 | 重建表格组件，统一表头、列宽、行高和对齐 |
| Acronym退回 | STAR/SWOT/OKR等缩写模型把字母、英文全称、中文解释拆散 | 重建术语记忆组件，绑定字母与英文全称，中文作为解释层 |
| Motion退回 | 动效没有内容目的、全页同一动画、影响讲师控场或导出 | 退回动效设计部重写动效脚本 |
| Optical Balance退回 | 页面头轻脚重、左重右轻、视觉重心漂移、主判断不是第一眼落点 | 退回Page Spec和版式策略，重建视觉重量 |
| Reveal State退回 | 多步骤页面默认露出第1点、答案或后续阶段；多个语义阶段一起出现 | 退回动效脚本和交互实现，补首态/后续态矩阵 |
| Asset退回 | 关键页依赖Logo/截图/案例但仍是占位 | 接入真实素材或改成非冻结样张 |
| Chrome退回 | 控件抢画面、截图带按钮、export不干净 | 按`06-interaction-standards.md`重做模式 |

## 执行步骤

1. 读取交付定义和UI方向卡，确认场景和审美关键词。
2. 读取或检索`15-visual-reference-library.md`，选择候选参考并完成《参考经验转译卡》。
3. 选择至少两个视觉方向，例如稳健专业型、叙事培训型、高冲击发布型；每个方向都必须说明与当前用户资料的绑定关系。
4. 为每个方向制作Style Frame视觉证据，覆盖字体、颜色、标题层级、组件、控件显示、留白版心和AI化风险。
5. 若涉及HTML演示或分步呈现，读取`17-motion-design-standards.md`，补动效脚本和静止页清单。
6. 若涉及design-dna或guizang，读取`20-design-dna-guizang-integration.md`，补Presentation Design DNA档案、DNA来源追踪表、设计引擎桥接表、UI吸收表和动效吸收表。
7. 若涉及stylekit/taste/frontend-slides/html-ppt/huashu-design、排版/UI不好看或动效单一，读取`21-design-skill-stack-integration.md`，补Design Skill Stack决策表、StyleKit吸收合同、TasteSkill吸收合同和Motion source map。
8. 按反AI味负面清单逐项排查。
9. 检查关键页是否有真实素材或明确标注样例性质。
10. 若结构不成立，先重写Page Spec，再做视觉实现。
11. 副总编审核部门意见真实一致后，总编确认一个方向，才能进入全稿生产。
12. 对用户点名丑页、失衡页、动效缺失页，逐页填写“退回类型、根因、不可用修法、Page Spec重写方向”。未完成时不得恢复生产。
