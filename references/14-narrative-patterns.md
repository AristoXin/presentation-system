# 14-narrative-patterns

读取时机：当需要根据项目类型选择叙事结构、重构章节、生成叙事蓝图或避免所有材料套用同一结构时读取。

## 原则

1. 叙事结构由项目类型、受众和使用方式决定。
2. 不把培训结构套到客户汇报、路演、研究报告或复盘材料上。
3. 可以组合骨架，但必须说明组合理由。
4. 每页只保留一个核心观点，证据服务于该观点。

## 用途到叙事结构

| 项目类型 | 推荐结构 |
| :--- | :--- |
| 客户汇报 | Executive Summary → Context → Key Findings → Implications → Recommendation → Next Steps |
| 培训课件 | Learning Goal → Concept → Example → Practice → Reflection → Tool |
| 路演BP | Problem → Market → Solution → Product → Traction → Business Model → Team → Ask |
| 内部复盘 | Goal → Facts → Gap → Root Cause → Decision → Action Owner → Timeline |
| 销售提案 | Pain → Diagnosis → Solution → Proof → Implementation → Commercial Terms → Next Step |
| 研究报告 | Question → Method → Data → Analysis → Insight → Conclusion → Appendix |
| 产品介绍 | Scenario → Problem → Capability → Value → Use Case → Proof → Call to Action |
| 工作坊材料 | Objective → Agenda → Input → Activity → Discussion → Output → Follow-up |
| 招投标材料 | Requirement → Response → Evidence → Delivery Plan → Team → Risk Control → Compliance |
| 会议讲稿 | Opening Claim → Context → Evidence → Story → Implication → Closing Ask |
