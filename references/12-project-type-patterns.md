# 12-project-type-patterns

读取时机：当需要识别项目类型、避免把材料误判为培训课件、或根据不同场景选择内容重点和交付物时读取。

## 判断原则

1. 先看用户明示用途；用户已说明用途时，以用户说明为准。
2. 用户未说明时，从关键词、受众、语气、资料结构、交付格式和使用方式判断。
3. 不得默认培训。培训只是项目类型之一。
4. 若存在多个可能意图，输出意图假设，并说明默认按哪一种处理。

## 误判修正

- 出现“学员/练习/讲师”才优先考虑培训；仅有“讲解/说明/介绍”不能自动判为培训。
- 出现“客户/方案/评审/下一步”优先考虑客户汇报或销售提案。
- 出现“数据/趋势/方法/结论”优先考虑研究报告。
- 出现“问题/原因/行动项/责任人”优先考虑复盘或内部管理汇报。
- 出现“融资/BP/投资人/商业模式”优先考虑投融资路演。
