# 03-delivery-routing

读取时机：当需要根据项目类型、使用场景和交付格式决定HTML、PPTX、PDF、截图包、讲者备注、讲义、素材包等交付路线，或需要判断某个Skill不可用时如何安装、内嵌、改边界或阻断时读取。

## 按交付格式路由

默认规则：用户只说“创建文件”“生成版本”“做一个版本”“输出成品”，且任务属于演示、汇报、培训、视觉叙事或HTML样张场景时，默认创建HTML文件。PPTX、PDF、截图包、讲义等只有在用户明确要求，或HTML完成后作为衍生产物时才进入路线。

| 目标产物 | 首选Skill | 审批后备选边界 | 验收重点 |
| :--- | :--- | :--- | :--- |
| 原生可编辑PPTX | ppt-master | PPTX page spec + HTML preview + 母版说明 | 元素可编辑、母版一致、备注可用 |
| HTML交互演示 | frontend-slides | 静态HTML或page spec | 快捷键、滚轮、目录、全屏、响应式缩放、presentation chrome低干扰 |
| PDF/讲义 | 文档/PDF工具链 | HTML print stylesheet + 导出说明 | 页面断点、页码、图表来源、打印清晰度、隐藏交互控件 |
| 截图包/证据包 | 浏览器截图/导出工具 | 手工截图清单 | 命名、版本、全页完整性、证据字段、export模式无chrome |
| speaker notes | 语言内容与叙事流程 | 逐页讲稿 | 每页讲述目的、转场、提示、问答 |

## 按项目类型路由

| 项目类型 | 优先检查 | 常见交付 | 关键风险 |
| :--- | :--- | :--- | :--- |
| 客户汇报 | 证据链、决策信息、执行建议 | PPTX/PDF/HTML | 空泛价值主张、缺下一步 |
| 培训课件 | 学习路径、练习、讲师备注、带走工具 | PPTX/HTML/讲义/notes | 只讲概念，无练习和应用 |
| 投融资路演 | 故事张力、商业逻辑、关键页 | PPTX/PDF | 数据散、Ask不清 |
| 内部复盘 | 事实、责任、行动项、时间表 | PPTX/PDF | 只复述过程，不形成动作 |
| 研究报告 | 数据、图表、来源、结论可信度 | PDF/PPTX | 来源不清、结论过度 |
| 产品介绍 | 场景、功能、用户价值、案例 | HTML/PPTX/PDF | 功能堆叠，缺场景 |

## 选择步骤

1. 根据`02-workflow.md`中的交付定义确认使用场景。
2. 若用户未指定格式，默认HTML；再判断是否需要后续编辑、只读归档或多格式衍生。
3. 判断是否需要讲述控制：需要讲者备注、目录、分步呈现时，将备注和交互列为必交付。
4. 对HTML演示定义模式：audience默认、presenter可显示工具、export隐藏所有chrome。
5. 检查相关Skill：读`08-skill-governance.md`，确认可用性、版本和阻断状态。必要Skill或控件不可用时，不得自动继续；必须先提请用户同意安装/启用或内嵌，用户拒绝、未授权或权限阻断时停止当前任务。
6. 记录交付路线：写明首选、备选、不可承诺项和验收重点。

## 降级口径

- `ppt-master`不可用：先暂停并提请总编和用户；用户同意后，才可不承诺原生可编辑PPTX，改交付HTML、PDF、截图包或结构化PPT设计稿。
- `frontend-slides`不可用：先暂停并提请总编和用户；用户同意后，才可不承诺完整Web演示体验，改交付静态HTML、PPTX或PDF。
- `taste-skill`不可用：先暂停并提请总编和用户；用户同意后，才可不宣称外部审美Skill校准，改按`11-aesthetic-anti-ai.md`人工审美闸门执行。
- `headroom`不可用：先暂停并提请总编和用户；用户同意后，才可使用上下文摘要、文件索引和版本日志。
- PDF导出不可用：先暂停并提请总编和用户；用户同意后，才可输出HTML print stylesheet和导出说明。
