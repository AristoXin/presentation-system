#### Design DNA 摘要

**Design System**

- Color：使用少量高识别色形成活动记忆，但必须给颜色分配角色，例如背景能量色、信息容器色、强调词色、警示/指向色和中性正文底。
- Typography：超大标题承担情绪和传播点，中号标题承担页面任务，小号注释/标签承担解释；标题可倾斜、描边或压缩，但正文必须保留可读行距。
- Layout：不是单页模板，而是跨页版式族谱。常见结构包括大标题封面、人物/商品抠图锚点、斜切信息带、双步骤卡、四层级策略栏、评论区文案Demo和底部状态条。
- Components：标题牌、撕纸边、引用符、箭头、便签、步骤卡、商品抠图、人物抠图、网格底纹、细小英文状态条、橙/蓝/黄强色块都必须有角色，不得随机撒。

**Design Style**

- Mood：年轻、社媒、行动感、节日营销、热闹但有系统。
- Genre：活动策划Deck + 社媒内容手册 + 商品种草Demo + 编辑部拼贴。
- Composition：通过跨页重复的网格和色带建立统一，通过每页不同的主视觉锚点和动作模块建立变化；信息不是平均摆放，而是按“先吸引、再解释、再示范、再行动”分层植入。
- Brand Voice：可以轻快、口语、带梗，但每页必须有可执行动作，不只是漂亮口号。

**Visual Effects**

- 动效不应只是淡入。适合的动作包括：贴纸/标题牌入场、箭头指向、步骤卡逐步揭示、商品抠图轻推、评论区Demo逐条出现、背景网格保持静止。
- 动效设计必须和语言内容共管：哪个内容先被看见、哪个动作由讲者触发、哪个商品或口播句承担转场，都要进入首态/后续态矩阵。
- PPTX/PDF或图片PPT路线必须保留静态可读版本；HTML路线可增加轻量动效，但不得让高饱和色块和贴纸遮挡正文。

#### 使用规则

1. 使用本条目前，必须先把用户提供的拼图拆成多个页面角色，不能把一张长图当作单页参考。
2. 品牌审美、版式/UI、语言内容、动效设计、交互/工程和QA都必须参与转译：UI负责版式系统，语言负责页面动作，动效负责揭示节奏，交互/工程负责HTML/PPTX/图片PPT路线，QA负责跨页一致和投屏可读。
3. 采用本条目时，至少产出《参考经验转译卡》《页面族谱》《内容到UI/动效转译表》《Motion source map》和《部门Skill使用矩阵》。
4. 若产物只是几个色块、几张卡片、若干图标和文字填充，必须判定为未吸收本参考；该参考的核心不是“热闹配色”，而是跨页campaign系统。
5. 对正式商务、研究、管理汇报类项目，默认只借鉴结构方法，不借用表层拼贴皮肤。

#### Design DNA 摘要

**Design System**

- Color：必须定义primary、secondary、accent、neutral、semantic、surface和contrast strategy，并映射到演示的调色板语义表。
- Typography：必须定义display、H1/H2/H3、body、caption、overline和heading/body/mono字体族，并映射到投屏字号和字重阶梯。
- Spacing/Layout：必须定义base unit、spacing scale、content density、grid、columns、gutter、alignment tendency，并映射到页面族谱。
- Shape/Elevation/Components：必须定义圆角、边框、分隔符、阴影、按钮/卡片/列表/导航等组件边界；若交付为PPT/PDF，需转译为静态组件语法。

**Design Style**

- Mood / genre / personality：必须来自用户资料、受众和场景，不得只来自参考图或模板偏好。
- Visual language：复杂度、装饰程度、留白策略、视觉重量、焦点策略和对比级别必须写入Style Frame协同输入。
- Composition：层级方法、平衡类型、阅读流向、分组策略和负空间角色必须进入版式/UI规则。
- Brand voice：UI文案语气、正式程度和CTA表达不得被风格系统改写到偏离项目立场。

**Visual Effects**

- Effects：Canvas、WebGL、3D、shader、SVG、scroll、cursor等效果必须先判断是否适合演示场景和导出格式。
- Performance：必须记录effect intensity、performance tier和fallback strategy。
- Degradation：HTML可实现动态效果；PPTX/PDF/截图包必须有静态可读降级，不得牺牲可读性和证据链。

#### 使用规则

1. 用户提供视觉参考、品牌规范、旧稿或要求深度审美设计时，品牌审美副主编必须判断是否调用PPT-DNA；不调用也要写理由。
2. 调用或采用本条目时，必须把三维DNA转译为《参考经验转译卡》《UI方向卡》《页面族谱》《动效脚本》和《审美QA记录》。
3. DNA字段必须服务当前内容关系。若某token、组件或效果无法说明来自用户资料、参考经验或交付约束，必须删除。
4. 只生成JSON、只列风格形容词、只写“设计系统已建立”但没有页面映射，必须标记为“已提取未吸收”。

#### Design DNA 摘要

**Design System**

- 风格A：衬线标题、非衬线正文、等宽元数据、暖色/墨水纸张主题、图片网格、hero与non-hero节奏。
- 风格B：无衬线、12列网格、单一accent、直角纯色、无渐变/阴影/圆角、字号越大越细、中文标题分档、投屏最小字号。
- Layout：A线10类布局；B线正文页锁定S01-S22，必须建立页码→data-layout→选用理由→图片槽位映射。
- Components：卡片、KPI、timeline、图片容器、caption、hairline、图标和动效recipe必须来自对应模板，不得跨风格混用。

**Design Style**

- 风格A偏编辑部、人文、行业观察、故事化演讲；风格B偏信息系统、技术/数据、工程/设计、发布和年度总结。
- 主题节奏比单页漂亮更重要：hero dark / hero light / light / dark 必须形成呼吸，连续同主题过长视为疲劳。
- 版式选择必须来自当前内容关系：数据页用数据版式，结构页用结构版式，图片页用图片槽位，不能用卡片矩阵通吃。

**Visual Effects**

- WebGL/ASCII背景只服务氛围，不得压过内容。
- Motion One动效必须按页面语义选择，不能全页统一fade-up。
- B键低功耗、export稳定态、截图等待动效完成和无动画可读降级必须进入交互/QA约束。

#### 使用规则

1. guizang只能在内容关系、页面族谱、主题节奏和模板适配性审查完成后作为主设计引擎或Style Frame实现边界。

#### Design DNA 摘要

**Design System**

- Source of truth：只使用 StyleKit 的 `references/style-prompts.json` 和脚本返回的 `style_choice`，不得编造不存在的 style slug。
- Token strategy：从 StyleKit 输出中抽取色彩、字体、间距、圆角、图表、组件和状态约束，再转译为演示页的主题变量、页面族谱、图表样式和视觉锚点。
- Composition：优先读取 `composition_plan` 中的 style、layout、motion、interaction 所有权；用于解决多风格混合时谁负责颜色、谁负责字体、谁负责空间节奏。
- QA：若输出含 `quality_gate`，必须读取 `violations` 和 `autofix_suggestions`；失败项不能被当成“可后续优化”。

**Design Style**

- Mood：由当前项目目标、受众和交付场景决定，不由 StyleKit 的候选风格单独决定。
- Genre：StyleKit 是风格库和决策引擎，不是模板库；它提供的是可执行约束、反泛化规则和候选方向。
- Composition：把前端语言转译为演示语言：page/section 转为页面族谱，component 转为页内视觉模块，state 转为分步呈现或讲述节奏，interaction 转为动效脚本或不适用说明。
- Brand Voice：保留用户语言和项目语气；StyleKit 输出只作为设计语法，不改写内容立场和业务判断。

**Visual Effects**

- Motion：StyleKit 的 interaction / motion 建议只能进入动效脚本草案，必须经过 `17-motion-design-standards.md` 的首态/后续态矩阵和动效美感门禁。
- Implementation：HTML演示可吸收部分 hover、focus、transition 规则；PPTX/PDF默认只转译为静态层级、强调顺序和必要的页内构图。
- Degradation：投屏、PDF、PPTX和截图包必须有静态可读版本；不得因为StyleKit风格建议牺牲文字对比、图表可读性和证据链。

#### 使用规则

1. 任务进入 Style Frame、视觉方向选择或用户要求“更好看/更有审美/参考风格”时，先在参考库检索记录中列出本条目是否采用。
2. 采用本条目时，必须保存或记录 StyleKit 检索输入、候选slug、采用/弃用理由、转译后的演示规则和QA状态。
3. StyleKit 输出不得直接作为最终提示词交给生产执行；必须先生成《参考经验转译卡》，说明哪些规则进入当前页面族谱，哪些前端语汇被弃用。
4. 若候选风格属于强娱乐、强童趣、强节日、强街头、强奢华或强暗色控制台，仍需按本库通用型/专用型规则判断，不能因StyleKit推荐而默认可用。
5. 使用多风格混合时，必须写明 color owner、typography owner、spacing owner、motion owner，避免同一份deck同时出现多套表层审美。
6. 正式商务、研究报告、培训课件和客户提案中，StyleKit只能帮助选择母线和约束，不能替代品牌审美副主编、版式/UI副主编和QA副主编的实现后复核。

2. 会议汇报和商务汇报默认只借骨架：大标题、编号、栅格、黑白卡片、稳定页脚。
3. 技术课程、产品发布、社媒传播可以借完整皮肤，但正文页必须降低纹理密度和色彩刺激。

#### Design DNA 摘要

**Design System**

- Color：浅灰白背景为主，黑色承载标题与核心识别，天蓝色作为会员卡主版本，白色/黑色作为版本变体，少量蓝点用于品牌状态提示。推荐实现值可从 `#F1F2F3`、`#FFFFFF`、`#050505`、`#72B5FF`、`#9CCBFF`、`#8D939A`、`#D7DADF` 起步。
- Typography：主标题使用粗体几何无衬线，圆润、现代、强品牌感；辅助信息使用等宽或类等宽小字、大写、宽字距；卡面文字可使用弧形排版和小字号，制造实体票券/徽章感。
- Layout：四页品牌识别Deck结构，包括品牌主张页、会员价值解释页、三版本展示页和限量招募CTA页；主视觉通常是大留白中的单个票券对象。
- Shape：圆角会员卡、侧边票孔、邮票齿孔、圆形徽章、柔和投影、细分隔线和胶囊CTA。形状语言强调“可持有、可收藏、可编号”。

- Mood：精致、轻奢、安静、可信、收藏感、社区归属。

- Genre：Premium Membership Identity + Soft Minimal Brand Deck + Collector Pass System。
- Composition：首屏用单张蓝色会员卡建立记忆；第二页用黑色票券和三点理由解释价值；第三页用三种版本展示视觉系统延展；结尾页用大号“first 100”完成稀缺性转化。
- Imagery：不依赖照片；会员卡、徽章、齿孔、柔影和版本物件就是核心资产。扩展时应补充权益图标、真实用户状态或会员权益示例。

#### 使用规则

1. 适合会员社区、Early Access、活动通行证、收藏型产品、数字身份和限量发售品牌。
2. “编号、first 100、claim your number”等稀缺表达必须有真实机制和权益支撑。
3. 票券/卡片隐喻适合身份和通行，不适合普通工具功能、技术架构或财务报告。
4. 浅灰低对比页面用于品牌展示很高级，但投屏、培训和长文场景要提高正文对比度。
5. 常规商务汇报、会议总结和合规材料默认禁用该表层风格，只借鉴版本展示、身份系统和CTA收束方法。

#### Design DNA 摘要

**Design System**

- Color：几乎单色的近黑背景承载全部页面，白色用于主标题和关键数字，深灰用于圆形仪表、卡片边框和辅助刻度，少量橙色只用于 armed/status 警示。推荐实现值可从 `#050607`、`#101112`、`#1D1F20`、`#F3F3EF`、`#A6A8A6`、`#55585A`、`#E36A3C` 起步。
- Typography：主标题使用宽松但强壮的现代无衬线，display 尺寸极大、字重高、行距紧；正文与状态信息使用小号等宽字、大写、宽字距；指标数字使用大号几何无衬线，单位以小字号紧贴。

- Shape：半透明矩形卡、细线分隔、radial dial、圆形刻度、极细边框、微弱角标和白色矩形CTA。形状语言偏航电仪表、授时系统和任务控制台。
- Components：系统名封面、圆形基准仪表、time/space/trust能力卡、精度指标条、经纬度/UTC状态、部署按钮、文档按钮、任务状态角标。

**Design Style**
