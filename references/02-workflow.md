# 02-workflow

读取时机：当任务从原始资料启动、需要定义项目意图/交付物/内容差距/UI方向，或用户提供模板但目标不清时读取。

## 标准流程

```text
若为Skill治理: 历史对话需求整合
↓
资料接收
↓
资料盘点与风险扫描
↓
项目类型、受众、使用场景识别
↓
创建《团队协作与生产记录》骨架
↓
检测subagent能力并建立Agent编制表
↓
语言内容、审美、UI/版式、动效设计、交互/交付部门并行初审
↓
部门一致意见达成
↓
副总编审核部门一致性、冲突和新增部门需求
↓
总编初审方案闸门
↓
交付成果定义
↓
内容核查与差距分析
↓
页数充分性判断、内容深挖和页面族谱
↓
叙事结构重构
↓
交付路线选择
↓
参考经验检索、趋势适用性和弃用判断
↓
模板/设计引擎适配性审查与改造差异表
↓
内容到UI/动效转译、UI方向和动效脚本定义
↓
Style Frame审美闸门
↓
首态/后续态矩阵与Style Frame截图复核
↓
能力域副主编生产前放行
↓
建立制作工单并下发执行层
↓
页面生产与多格式输出
↓
执行层提交与副主编一级复核
↓
执行层修复与副主编复审循环，直至全部副主编一致同意
↓
总编二级复核
↓
总编退回意见下发副主编，副主编下发执行层，循环至总编无意见
↓
QA验收与交付确认
↓
提交用户验收
↓
用户退回时回到总编分发与副主编复审闭环
```

## 生产前必须完成

1. 《项目类型判断卡》
2. 《受众与使用场景卡》
3. 《原始资料盘点表》
4. 《交付定义卡》
5. 《内容差距清单》
6. 《叙事结构建议》
7. 《UI方向卡》
8. 《交付格式与降级路径》
9. 《团队协作与生产记录》文件，内含《RACI矩阵》《Agent编制表》和《制作工单》
10. 《部门一致意见表》，至少覆盖语言内容、品牌审美、版式/UI、动效设计、交互/交付
11. 《副总编一致性审核记录》和《总编初审方案记录》，结论为通过/退回；退回时必须指向对应副主编
12. 《Skill调用决策表》，标明每个相关Skill为“调用/可用但不调用/需安装/需内嵌/降级”
13. 《参考经验转译卡》
13a. 《正向参考图深拆与经验总结表》，仅当采用正向参考图时必填
14. 《内容深挖表》《页数充分性判断表》和《页面族谱表》，说明用户内容如何生成页数、页面角色和拆合页策略
15. 《参考库/趋势适配与否决记录》，列候选、采用、弃用和弃用理由
16. 《部门联合审查表》，各部门必须有独立判断、反对意见、放行/否决和subagent id
17. 《内容到UI/动效转译表》，说明每页内容关系如何转成UI结构、视觉锚点、动效或静止策略
18. 《模板适配性审查表》，涉及任何模板、历史样张、Skill版式或设计引擎时必填
19. 《模板改造差异表》，模板适配通过后必填；未通过则写弃用原因
20. 《禁止硬套自检表》，确认没有先选风格、模板填空、主Agent自签
21. 《Style Frame部门协同输入表》，语言内容、品牌审美、版式/UI、动效设计、交互工程、QA前置风险先输入并解决冲突
22. Style Frame审美确认记录，必须绑定协同输入表和截图证据
23. 生产前能力域放行结论：语言内容、品牌与审美、版式/UI组件、动效设计、交互/多格式、工具链至少各有“通过/退回/暂不适用”判断

若项目涉及模板，还必须完成《参考模板解析表》和《页数预算表》。若任何部门未达成一致，不得提交总编复核；必须先由相关副主编修订初步方案并重新记录一致意见。若总编初审方案未通过，不得进入Page Spec、Style Frame、制作工单或执行层。

生产前硬阻断：若《团队协作与生产记录》尚未创建，或其中RACI、Agent编制表、制作工单、总编目标、验收标准、证据要求为空，不得创建/修改HTML、PPTX、PDF、截图包或主交付文件。若是在既有版本上修复，也必须先补齐该版本的协作记录，再修改产物。子Agent、必要部门、工具不可用时，不得直接降级；必须写入`SUBAGENT-BLOCKER`并停止当前生产/复核链路。

## 操作步骤

1. 盘点资料：列出资料类型、完整度、结构、数据可信度、敏感信息、品牌资产、视觉素材和交付约束。
2. 识别项目类型：先读`12-project-type-patterns.md`，从客户汇报、培训、销售提案、路演、研究报告、产品介绍、内部复盘等类型中判断，不默认培训。
3. 识别受众与使用方式：客户、高管、员工、学员、投资人、合作方、内部团队；现场讲、线上发、打印、归档、二次编辑、投屏、网页展示。
4. 建立团队协作骨架：读`01-team-roles.md`和`16-multi-agent-governance.md`，创建或更新《团队协作与生产记录》文件，输出本项目RACI矩阵和Agent编制表，明确总编、主责副主编、执行专员、QA副主编和记录专员。
5. 并行初审：由语言内容、品牌审美、版式/UI、动效设计、交互/交付等部门subagent分别提出初步意见；记录一致项、冲突项和修订动作。所有适用部门未一致通过时，不得提交副总编。
6. 副总编一致性审核：副总编检查部门是否真实独立、是否存在冲突、是否需要新增部门、是否有降级风险。副总编未通过时，退回对应副主编；通过后才提交总编。
7. 总编初审方案：总编只接收副总编通过的初步方案；通过后只允许进入交付定义、页数充分性、Page Spec和Style Frame等生产前设计阶段，退回时必须指向对应副主编重新协商。制作工单不得早于交付定义、页数充分性、页面族谱、内容到UI/动效转译、模板适配/改造和Style Frame协同输入。
8. 定义交付：明确项目类型、核心目标、受众、场景、演示者、格式、页面密度、互动要求、审美方向和禁止事项。若用户未指定格式，按`03-delivery-routing.md`判断交付路线。
9. 查内容差距：判断目标所需内容是否齐全；缺失内容可提炼、假设补写或列入待补充。
10. 判断页数充分性：不能默认沿用用户旧版、样本、模板或历史产物页数。由需求/交付、语言内容、品牌审美、版式/UI、动效、交互/交付和QA共同判断当前页数是否足够，并输出增页、拆页、合页、删除、附录或讲者备注承载建议。
11. 重构叙事：先读`14-narrative-patterns.md`，按项目类型选择叙事结构，输出章节、每页核心观点、证据、页面功能和预期受众反应。
12. 内容深挖：填写《内容深挖表》，明确用户目标、关键案例、训练动作、认知负荷和不可被模板压扁的部分。
13. 页面族谱：填写《页面族谱表》，为每页定义页面角色、内容功能、视觉形态、讲述动作、动效意图和“不能长成什么样”。页面族谱必须承接页数充分性判断，不能把多个应拆分的页面角色硬塞进一页。
14. 检索参考经验：涉及视觉设计时先读`15-visual-reference-library.md`，选择1个主参考方向和最多1个辅助参考方向；必须说明哪些经验可借鉴、哪些表层元素不能照搬，并列出弃用参考和弃用理由。若采用正向参考图，必须填写《正向参考图深拆与经验总结表》，回答配色、文字协同、字体系统、版面规划、页面切换、替代方案、动效和HTML交互状态机。
15. 转译当前项目：把参考经验改写成当前项目的主品牌色、受众、观看距离、内容证据、页面族谱和交付格式；若当前资料不足以支撑参考风格，必须退回重定向，不得用“降级”绕过。
16. 部门联合审查：内容、UI、审美、动效、参考库、流行调研、交互、QA分别给出通过/退回/否决。任一部门否决，不得提交副总编。
17. 内容到UI/动效转译：语言内容副主编先与版式/UI、动效设计共同填写《内容到UI/动效转译表》，把每页核心判断、证据关系、讲述动作转成UI结构、视觉锚点、动效或静止策略。未完成时不得定义UI方向。
18. 模板适配性审查：涉及任何模板、历史样张、设计引擎、Sxx版式或主题风格时，先填《模板适配性审查表》。证明不适合时必须弃用；证明适合也不得直接生产。
19. 模板改造差异：候选模板通过适配后，逐页填写《模板改造差异表》，说明保留、改写、新增、删除和调整项。没有该表，不得复制模板、生成HTML/PPT或进入Style Frame。
20. 禁止硬套自检：填写《禁止硬套自检表》，若存在先选风格、模板填空、主Agent自签或模板无法承载仍使用，不得进入生产。
21. 定义UI：输出主色、辅助色、字体、密度、组件、图表表格风格、交互和适配要求；UI方向卡必须引用《参考经验转译卡》《内容到UI/动效转译表》《模板适配性审查表》和《模板改造差异表》。
22. 定义动效：读取`17-motion-design-standards.md`，输出动效脚本、静止页清单、分步呈现规则、导出/演讲风险；动效脚本必须引用内容讲述动作和UI结构，不得独立发明动画。
23. Style Frame协同生成：完整视觉生产前先读`11-aesthetic-anti-ai.md`，填写《Style Frame部门协同输入表》，解决语言内容、品牌审美、版式/UI、动效设计、交互工程、QA之间的冲突，再保留样张截图或清晰版式证据。
24. 生产前放行：语言内容、品牌与审美、版式/UI组件、动效设计、交互体验、工具链/多格式副主编在协作记录中写入通过/退回结论；退回时不得进入生产。
25. 执行与一级复核：执行层产出后，各能力域副主编必须填写一级复核记录；退回项由副主编subagent生成修复任务并下发执行层。
26. 多轮复审循环：执行层修复后回到同一副主编subagent复审；任意一项未通过，继续执行“退回-修复-复审”，不得直接交副总编或总编。每轮必须记录轮次、责任副主编、执行层修复证据和复审结论。
27. 副主编一致同意：全部适用副主编subagent签署“可交副总编”后，副总编复核一致性；副总编通过后总编才能复核宏观目标、冲突取舍和是否进入QA。
28. 总编退回复核：总编提出新意见时，必须下发给对应副主编subagent；副主编再下发执行层，执行层完善后回副主编复审，再由所有适用副主编一致同意、副总编通过后重新交总编。
29. 复核关键页：若是既有HTML/PPT迭代，先按`04-design-standards.md`的HTML审美审计模板检查重复版式、文字密度、占位素材和控件状态。
30. 选择交付路线：读`03-delivery-routing.md`并检查相关Skill。
31. 用户验收：QA冻结后提交用户验收；用户退回时，总编必须把意见分发给对应副主编，重新进入副主编复审循环和QA回归。

### 页数充分性判断表

任何正式视觉生产、重做或恢复生产前都必须填写。该表不只用于模板任务；即使用户没有给模板，也必须判断页数是否足够。

| 字段 | 结论 |
| :--- | :--- |
| 现有参考页数 | 用户旧版/样本/模板/历史产物页数；若无则写无 |
| 页数约束等级 | 用户写死/强建议/仅参考/无约束 |
| 内容部门判断 | 是否有多个核心观点被挤在同页；是否需要拆页 |
| 文案部门判断 | 标题、解释、案例、讲者口径是否需要独立页面或备注 |
| 审美部门判断 | 当前页数是否支持留白、节奏、视觉锚点和观看舒适度 |
| 版式/UI判断 | 当前页数是否导致组件拥挤、表格过密、左右栏失衡或卡片泛滥 |
| 动效部门判断 | 分步揭示、演示节奏和互动动作是否需要独立页面 |
| 交互/交付判断 | 是否需要附录、讲者备注、隐藏层、备份页或证据包 |
| QA判断 | 若按当前页数生产，可能触发哪些硬失败 |
| 建议最终页数 |  |
| 新增页 | 页码/页组、页面角色、原因 |
| 拆分页 | 原页、拆分后页面、原因 |
| 合并/删除页 | 页码、原因 |
| 附录/备注承载 | 哪些内容移入附录、讲者备注或证据包 |
| 总编结论 | 采用/退回/提请用户确认 |

## 阻塞条件

- 用户明确要求固定页数但内容明显超载：先说明风险，提出主体页和附录拆分。
- 资料用途无法判断：只能输出诊断文本、意图假设和Page Spec草案；不得创建或修改HTML/PPT/PDF/截图包/Vx目录，也不得把默认路径输出称为生产初稿。
- 涉及客户敏感信息或版权不明素材：标记风险，要求脱敏或替换。
