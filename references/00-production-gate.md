# 00-production-gate

读取时机：任何生产、修复、回退、审稿、QA冻结、用户退回回归、Style Frame生成、HTML/PPT/PDF/截图包创建前必须先读取。本文件必须与`19-approved-production-flow.md`共同作为正式执行总闸门；其他reference只能追加要求，不能降低本文件和用户审核流程图的门槛。

## 用户审核流程图

执行本文件前必须先读取`19-approved-production-flow.md`。该流程图是当前Skill内强制流程，任何生产、修改、回退、失败停产或恢复生产，都必须沿流程图执行；不得用“小修”“局部改一下”“主Agent直接修复”绕过副主编审核、执行层修订、同一副主编复审、全部适用副主编一致审核、副总编一致性审核、总编二级审核、QA回归和用户复验。若用户最新要求与旧流程图冲突，以《用户最新要求锁定表》为更高优先级，并先修订流程再生产。

## 先判定任务等级

优先级硬规则：Skill治理、失败治理、流程质疑和用户最新要求锁定优先级高于“小修”“新版本生产”和“继续优化”。同一请求同时包含“生成新版本”和“补/修Skill”时，先按Skill治理处理；未完成治理修订、Subagent Evidence Lock和用户明确恢复生产前，不得创建或修改任何Vx产物。

| 等级 | 场景 | 最低门槛 |
| :--- | :--- | :--- |
| 诊断/审计 | 不改交付物，只判断问题或修Skill | 可不生成Style Frame，但必须记录审计对象、证据和结论；若修Skill，先按`18-history-derived-governance.md`整理历史对话需求 |
| 单页/小修 | 修1-2页、修控件、修明显bug | 受影响能力域副主编Agent + QA副主编Agent；必要时补单页Style Frame或Page Spec |
| 正式视觉生产 | 新建或重做HTML/PPT/PDF/截图包 | 完整多Agent、内容到UI/动效转译、至少2个Style Frame方向、QA冻结 |
| 用户审美退回 | 用户说丑、像模板、没审美、未调用subagent/Skill | 暂停生产；先失败审计、旧版否决、审美趋势/经验检索、至少2个Style Frame方向 |
| 失败停产 | 自动验证通过但审美失败、用户退回、QA阻断、证据不足却曾宣称完成 | 标记`FAILED_VISUAL_FREEZE`；只能做失败审计、证据归档、门禁修订和Style Frame候选；恢复生产需用户明确批准 |
| 完整Skill审计 | 用户要求完整检查/全面更新Skill，或质疑规则写了但没有在下一版生效 | 只修改Skill系统文件和治理证据；输出完整Skill审计矩阵、规则闭环映射和验证记录 |

## 不可跳过检查

| 闸门 | 必须完成 | 阻断条件 |
| :--- | :--- | :--- |
| 任务等级 | 明确诊断/小修/正式生产/审美退回 | 未分级 |
| 用户最新要求锁 | 生产/修复/QA/Skill治理前完成《用户最新要求锁定表》，忠实转写用户原话、禁止事项、验收标准和冲突项 | 主Agent自行解释用户要求；未记录“按流程图/subagent/结果不满意”等最高优先级要求 |
| Vx版本原子门禁 | 创建、命名、复制、修改、导出、截图验证或归档任何Vx/HTML/PPT/PDF/截图包前，完成Subagent Evidence Lock | 用“未冻结/样张/小修/候选/工程烟测”解释为何可以先动产物 |
| Subagent Evidence Lock | 真实subagent id、能力域、工具/线程证据、创建时间、输入路径、输出结论齐全 | 无真实agent证据；主Agent代签；只有RACI表；用`SUBAGENT-BLOCKER`冒充可放行证据 |
| 历史需求整合 | Skill治理前完成《历史对话需求整合表》 | 直接读取现有Skill文档并按文档自证 |
| 协作记录 | 创建《团队协作与生产记录》并写入版本、目标、RACI、Agent编制表；主文件创建时间不得早于记录和subagent证据 | 产物早于记录；Vx目录早于Agent编制表 |
| subagent | 创建真实能力域Agent；不可用时写`SUBAGENT-BLOCKER`并停止当前生产/复核链路 | 主Agent分角色、无Agent id、无真实Agent证据 |
| 预审与复核分离 | 生产前初审、执行后一级复核、退回后同一subagent复审分别记录 | 用部门一致意见、方案预审、总编初审、主Agent自查冒充实现后复核 |
| 修改/回退一致审核 | 同一副主编复审后，全部适用副主编重新一致审核，再交副总编一致性审核 | 只由打回副主编同意后继续；跳过其他适用副主编 |
| 内容转译 | 《内容到UI/动效转译表》 | UI凭模板或审美直接排版 |
| 页数充分性 | 《页数充分性判断表》，含内容、文案、审美、UI、动效、交互/交付、QA判断 | 默认沿用旧版/样本/模板页数；用缩小字号、压缩文案、密集卡片消化页数不足 |
| 审美经验 | 《参考经验转译卡》；需要当前趋势时完成《主流审美趋势调研卡》 | 套旧模板、未说明不照搬项 |
| Design DNA深度集成 | 《Presentation Design DNA档案》《DNA来源追踪表》《UI吸收表》《动效吸收表》 | 只有DNA JSON、形容词、参考名或风格标签；DNA没有落到页面族谱、组件、动效和QA |
| Design Skill Stack集成 | 《Design Skill Stack决策表》《StyleKit吸收合同》《TasteSkill吸收合同》《Motion source map》 | 只列Skill名称；只写slug/prompt；stylekit/taste/frontend-slides/html-ppt/huashu-design可用但无调用/弃用理由 |
| 联网搜索/外部参考 | 用户要求联网搜索、当前趋势、优秀案例、最新文档，或审美退回需要外部验证 | 完成`23-online-research-toolchain.md`要求的Online Research Stack决策表、外部参考搜索记录、外部视觉参考转译卡、最新文档检索记录或工具阻断记录 | 用户要求联网但未搜索；工具不可用却假装搜索；只用本地参考库冒充外部趋势 |
| 缺失控件安装提请 | Tavily/Brave/Firecrawl/Context7/Playwright等必要控件缺失 | 写入《缺失控件安装提请记录》并获得用户同意后安装/启用；用户拒绝则停止 | 未提请用户就降级；用户拒绝后继续执行依赖该工具的Skill路径 |
| 参考库检索 | 《参考库检索记录》，含检索关键词、候选REF、弃用REF、采用REF、页面族谱映射 | 只说参考过经验库，没有检索证据；没有弃用REF和弃用理由 |
| 趋势校准 | 品牌审美副主编判断是否需要外部近期验证；需要时搜索，不需要时写理由；必须说明哪些趋势不适合当前内容 | 用户要求趋势但未搜索，或不搜索无理由；只罗列流行趋势不判断适配性 |
| 深度定制顺序 | 语言内容、品牌审美、版式/UI、动效、交互工程、QA先完成部门输入和一致意见，再选择设计引擎 | 先宣布风格、主题色、模板或Sxx版式，再倒推内容结构 |
| 模板适配性 | 候选模板/Skill版式必须通过适配性审查，证明适合当前受众、证据、训练动作、讲述节奏和品牌语气 | 用户提供内容后直接套模板；模板不适合仍硬套；为适配模板而扭曲内容 |
| 模板改造 | 《模板改造差异表》，含保留/改写/新增/删除/调整项和责任部门 | 模板适配后直接调用；只改颜色文字，不改内容结构和页面组件 |
| Skill治理范围 | 只修改`SKILL.md`、`AGENTS.md`、`agents/openai.yaml`、`references/*.md`和治理证据 | 修Skill时创建或修改HTML/PPT/PDF/样张目录 |
| 完整Skill审计 | 逐项检查入口触发、流程阻断、部门职责、QA证据、版本证据、经验回写、元数据和验证脚本 | 只补某一个入口规则；未检查reference之间是否闭环 |
| 设计引擎 | `guizang-ppt-skill`/`html-ppt`/`frontend-slides`/客户母版之一，并有设计引擎桥接表和页码映射，且映射必须承接已经完成的内容到UI/动效转译和Presentation Design DNA | 只借风格名，无DNA桥接、layout/theme/动画映射；模板替代部门判断 |
| Style Frame | 方向数、协同输入、截图、时间戳、签署齐全 | 先截图后补表、单部门样张、冲突未解决 |
| 动效 | 动效脚本、静止页、export稳定态 | 工程层临时加动画、截图中间态 |
| 分步状态 | 首态/后续态矩阵；触发次数、初始态、完成态与脚本一致 | 第1点默认显示、阶段无理由合并、完成态在演示模式提前外露 |
| 光学平衡 | 重点页上下/左右重量、主视觉落点和阅读路径 | 头轻脚重、左重右轻、底部堆积、主视觉漂移 |
| QA | QA第0步先查真实subagent证据和同一副主编实现后复审；通过后才允许截图、Playwright、validation和人工审美复核 | 无subagent时继续工程烟测；生成validation；写工程烟测通过 |
| 实现后复核 | 真实产物生成后，原责任副主编subagent读取文件/截图/validation并签署一级复核；退回后同一副主编复审 | 只有生产前预审；主Agent自查；换人代签；无截图证据 |
| 失败阻断 | 审美退回、截图无效、工程烟测冒充QA、用户明确不满意或质疑流程/subagent时写入`FAILED_VISUAL_FREEZE`并执行产物冻结锁 | 继续生成下一版全稿；复制旧版；截图验证；把失败稿改名为候选稿；未获用户批准恢复生产 |
| 用户验收 | QA冻结后提交用户并记录结论 | 未验收称完成 |
| 经验回写 | 《经验回写记录》，含成功经验、失败经验、适用边界、关联版本 | 视觉项目未回写却冻结 |

## Style Frame等级

| 场景 | 方向数 | 覆盖范围 | 证据 |
| :--- | :--- | :--- | :--- |
| 单页/小修 | 1个方向；仅非视觉文案/结构小修可用Page Spec替代截图，需总编记录原因 | 受影响页 | 修前/修后截图或Page Spec；涉及UI、动效、控制条、光学平衡或截图干净度时必须有修前/修后截图 |
| 正式视觉生产 | 不少于2个方向 | 至少覆盖封面、核心模型/工具页、案例页、收束页中的3类 | 每方向截图 + 协同输入表 |
| 用户审美退回 | 不少于2个方向，且必须明显不可逆改变 | 旧版问题页 + 新版关键页 | 旧版否决、趋势/经验转译、截图对照 |

截图生成时间必须晚于《内容到UI/动效转译表》《Style Frame部门协同输入表》《动效脚本》和冲突解决记录。否则不得追认为Style Frame。

| 审计域 | 必查文件 | 必查问题 | 修订判断 | 验证方式 |
| :--- | :--- | :--- | :--- | :--- |
| 入口触发 | `SKILL.md`、`agents/openai.yaml` | 用户原话是否会触发正确模式；是否误触发产物生产 | 需改/无需改/阻断 | frontmatter、OpenAI yaml、路由表检索 |
| 强制流程 | `19-approved-production-flow.md`、`00-production-gate.md` | 失败后是否禁止继续生成；恢复生产是否需用户批准 |  | Mermaid流程和状态口径检索 |

## 状态命名硬门禁

状态名只能描述证据成熟度，不能反向授权创建产物。缺少真实subagent证据时，不得出现`Vx候选稿`、`Vx工作稿`、`Vx样张`、`Vx工程烟测通过`等状态；只能记录为`BLOCKED_BY_SUBAGENT`、`FAILED_VISUAL_FREEZE`、失败审计或文字Page Spec。

| 状态 | 允许用途 | 禁止措辞 |
| :--- | :--- | :--- |
| 草案/Page Spec | 描述内容结构、页面角色、实现建议 | 交付、完成、冻结 |
| 视觉草图/Style Frame候选 | 供审美方向讨论和用户批准恢复生产；仅在subagent证据齐全后可生成视觉文件 | 全稿、正式版、QA通过；无subagent时生成HTML/PPT/PDF/截图 |
| 样张/工作稿 | 已通过Subagent Evidence Lock后，内部验证局部结构或交互 | 无subagent时使用Vx编号；Vx已完成、交付完成、正式交付 |
| 候选稿 | 真实subagent初审、Style Frame、执行后复核完成，但尚缺用户验收或部分冻结条件 | 无subagent时称候选稿；正式交付、最终版 |
| 冻结稿 | QA已冻结但用户尚未验收 | 已验收、最终交付 |
| 正式交付 | 实现后subagent复核、总编二级复核、QA冻结和用户验收全部完成 | 无 |

用户验收尚未完成时，只能进入“候选交付/冻结待验收”，不能写“正式交付”。版本目录存在不等于版本已交付；README、最终回复和版本索引必须使用同一状态。

## 生产前放行

只有同时满足以下条件，才可进入文件创建、版本创建、复制旧版、全稿生成、截图导出、工程烟测或validation生成：

1. 真实subagent证据完整，记为`SEL-real`。
2. 内容、审美、UI、动效、交互工程、QA已完成生产前输入和冲突解决。
3. 页数充分性判断、内容深挖和页面族谱已完成；设计引擎、模板、主题色和版式编号的选择晚于部门输入和内容到UI/动效转译完成时间。
4. Presentation Design DNA档案、DNA来源追踪表、UI吸收表、动效吸收表和设计引擎桥接表已完成；若使用guizang，A/B适配与弃用表、主题节奏、逐页layout/Sxx映射、图片槽位和模板改造差异已完成。
5. Design Skill Stack决策表已完成；StyleKit候选/采用/弃用/quality_gate、TasteSkill Design Read/三拨盘/pre-flight、演示引擎选择和Motion source map已完成或写明工具阻断。
6. Online Research Stack决策表已完成；外部参考搜索记录、外部视觉参考转译卡、最新文档检索记录、Playwright视觉验证计划/记录或联网工具阻断记录已完成。必要控件缺失时，必须有安装提请记录和用户结论；用户拒绝安装/授权时不得进入生产前放行。
7. 《模板适配性审查表》《模板改造差异表》和逐页Page Spec完成；模板改造差异表晚于模板适配性审查，早于执行层生产。
8. 《首态/后续态矩阵》已覆盖所有分步页，且晚于内容到UI/动效转译、早于动效实现。
9. Style Frame截图晚于协同输入完成时间；失败审计期未获用户批准Style Frame探索时，只能提供文字Page Spec或版式草图说明。
10. 制作工单晚于以上条件，且写明交付物、关键受众、约束、主责副主编、执行专员、验收标准和证据要求。

## 冻结/交付放行

只有同时满足以下条件，才可进入QA冻结、候选交付、正式交付或最终回复中的“完成/交付”措辞：

1. 执行层真实产物晚于制作工单，且每个适用副主编有实现后复核记录；所有退回项已回到同一副主编复审。
2. QA有全页截图、contact sheet、重点页放大人工复核、validation结构字段、光学平衡/分步状态/动效美感复核和冻结结论；validation只能作为工程烟测证据。
3. 视觉项目QA冻结前完成经验回写；未回写时只能标为未冻结或样张。
4. 用户验收记录存在或明确标注尚未验收；尚未验收时不得称正式交付。
5. 缺少`SEL-real`的产物不得进入正式冻结或正式交付。

最终回复放行口径：

- 生产前放行和冻结/交付放行全部满足，且用户验收通过：可写“正式交付完成”。
- 缺少实现后复核或QA冻结但已通过Subagent Evidence Lock：只能写“工程烟测通过，未冻结/未交付”。
- 缺少真实subagent证据：只能写`BLOCKED_BY_SUBAGENT`或`FAILED_VISUAL_FREEZE`，不得写工程烟测通过，不得生成validation。
- QA冻结但用户尚未验收：只能写“冻结待验收/候选交付”。
- 用户已经退回审美或流程：只能写“当前版本未通过验收，已进入失败审计/Skill治理”，不得继续称为交付。
