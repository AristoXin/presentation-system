# 01-team-roles

读取时机：当任务涉及团队结构、职责分配、RACI、质量责任、跨岗位协作、问题追责或工作流治理时读取。任何正式生产、审稿、修复、QA冻结或Skill治理任务，也必须读取本文件。

本文件是生产系统的协作内核，不是附属说明。本文中的“总编”和用户口语中的“主编”指同一决策角色：负责最终方向、复核和退回，但不得替代副主编一级复核和QA冻结。副总编负责在总编前审核部门意见是否真实独立、一致且无被压制的反对意见。总编可以由主Agent担任。多Agent可用时，必须按`16-multi-agent-governance.md`拆分给独立Agent；不可用时，不得自动降级，必须写入`SUBAGENT-BLOCKER`并停止当前生产/复核链路。

硬门禁：任何正式生产、修复、审稿或QA冻结任务，必须先创建或更新本版本《团队协作与生产记录》文件，再进入工作平台试生产或正式生产。该文件不是最终说明附件，而是生产控制台；每个阶段的“部门会议、初始备忘录、关键页试生产、执行、一级复核、退回、修复、二级复核、冻结/未冻结”必须随着工作推进实时更新。若记录不存在，或RACI、Agent编制表、部门会议记录、交付文件初始备忘录、关键页试生产记录、工作平台记录、正式全稿制作工单、一级复核记录、总编二级复核记录、QA冻结记录为空，执行专员不得创建/修改最终交付物，QA副主编不得签署通过。

## 三级架构

| 层级 | 角色 | 核心使命 |
| :--- | :--- | :--- |
| 决策端 | 总编 | 定最终目标、交付路线、审美方向和验收标准 |
| 决策端 | 副总编 | 审核部门一致性、冲突协调、新部门申请和是否可提交总编 |
| 管理端 | 能力域副主编 | 负责诊断、叙事、审美、版式、交互、工具链、QA等能力域 |
| 临时交付端 | 工作平台 | 平时不启用；关键页阶段晚于初始备忘录启用，正式全稿阶段晚于副主编关键页初审/复审和总编初审同意启用；每次抽调各部门员工完成实现、修复和证据打包 |
| 执行端 | 专业专员 | 按工单执行生产、修复、测试和证据记录 |

## 协作原则

1. 决策权与执行权分离：总编定方向，副主编定标准并一级复核，执行专员落地。
2. 每个关键事项只有一个A；可以多人参与，但不能多人共同背最终责任。
3. 产出必须经过“部门会议 → 交付文件初始备忘录 → 工作平台关键页试生产 → 副主编初审/复审循环 → 全部副主编一致同意 → 总编初审/复审循环 → 工作平台正式生产 → 全部副主编实现后复核 → 副总编一致性审核 → 总编二级复核 → QA冻结 → 用户验收”。
4. 发现问题必须退回对应责任链，不得由QA直接替执行层修补后放行。
5. 用户点名任一问题页，问题页复盘副主编必须启动专项复盘，并要求跨页一致性专员检查同类页。
6. Skill或工具输出只是工作平台产物，不是最终稿；对应能力域副主编必须审查，总编必须复核，QA必须冻结。
7. 如果运行环境支持subagent，必须让需求/交付、语言内容、品牌审美、版式/UI、动效设计、交互工程、QA相互独立审查，避免单一视角自证通过。
8. 团队协作记录必须先于产物生产创建，并按阶段增量更新；不得在完成产物后一次性倒填。
9. 副主编一级复核必须独立于执行产出；总编二级复核必须基于副主编结论；QA冻结必须基于全页证据和人工视觉复核。
10. 发现需要新增部门时，只能提交《新增部门申请》；总编批准并创建独立subagent后，该部门才算成立。主Agent不得临时扮演新部门。
11. 工具、子Agent或必要部门不可用时，必须暂停正式生产，写入`SUBAGENT-BLOCKER`；不得以用户未拒绝、总编口头同意或主Agent分角色为理由继续。
12. 真实产物生产必须由工作平台接收制作工单后执行。工作平台不是常设部门，不能提前介入定标准；也不能在完成产物后自审通过。未启用工作平台时，主Agent或单个执行专员不得直接生成HTML/PPT/PDF/Vx产物。
13. 每个部门必须下设1-2名员工Agent或执行专员。部门会议必须由副主编和本部门员工共同完成，只有副主编与全部参与员工一致同意后，部门意见才可写入《交付文件初始备忘录》。
14. 所有subagent和部门员工必须按职责拥有中文名称，不得使用“Agent1”“审查员A”“worker”等泛名；中文名称必须写入Agent编制表、部门会议记录、工作平台编制表和复核记录。

## RACI定义

- R Responsible：直接执行。
- A Accountable：最终负责，只能有一个。
- C Consulted：参与建议，必须在关键决策前被征询。
- I Informed：同步知情，不阻塞执行。

## 闭环工作流

只读审计不进入本闭环，不创建生产协作记录；只需在输出中列明审计对象、证据路径和结论。需要写入Skill或产物时，必须重新进入Skill治理或生产闭环。

1. 协作记录初始化：记录专员先创建《团队协作与生产记录》，填入版本、任务等级、RACI、Agent编制表、验收标准和证据要求；没有记录不得启动生产性诊断或产物生产。
2. 启动诊断：总编主持，需求诊断副主编把项目意图、交付定义和页数充分性判断写入协作记录。
3. 工具链确认：工具链副主编检测Skill、vendor、版本、license和降级审批状态，并写入Skill调用决策表。
4. 内容重构：语言内容副主编输出内容差距、叙事蓝图、术语表达规则和内容到UI/动效转译表。
5. 视觉与动效定义：品牌审美副主编、版式/UI副主编、动效设计副主编先输出参考经验转译卡、UI方向、页面族谱、Style Frame部门协同输入、动效脚本和反AI味审查。
6. 部门会议：各能力域副主编组织本部门1-2名员工，根据用户文件、初始稿件、旧稿、此前生成稿件和失败证据形成部门意见；副主编与员工必须一致同意。
7. 交付文件初始备忘录：记录专员汇总各部门会议结论，形成《交付文件初始备忘录》，并写明冲突解决、关键页目标、关键页验收标准和不可继承旧问题。
8. 工作平台关键页试生产：工作平台协调员从每个适用部门抽调至少1名员工，按初始备忘录生产关键页/关键片段，不得直接生产全稿。
9. 副主编关键页初审：各副主编审查关键页；任一副主编退回时，工作平台修改后回到同一副主编复审，直到全部副主编一致同意“可交总编初审”。
10. 总编初审/复审：总编不同意时，退回工作平台修改，并重新经过对应副主编初审、全部副主编一致审核和总编复审；总编同意后才进入正式全稿制作工单。
11. 工作平台正式生产：总编初审同意后，工作平台按正式制作工单生产全稿；每个适用部门必须有成员参与，缺席即阻断。
12. 能力域实现后复核：正式全稿完成后，对应副主编审查工作平台产物；发现问题时由副主编subagent建立修复任务并下发工作平台，修复后回到同一副主编subagent复审。
13. 副主编一致同意：语言内容、品牌审美、版式/UI、动效设计、交互/交付、QA前置风险等适用副主编全部签署“可交副总编”后，副总编审核一致性；副总编通过后才能进入总编二级复核。
14. 问题页复盘：用户点名页或QA重大问题进入专项复盘，并同步检查同类页。
15. 总编二级复核：总编只在副主编一致同意和副总编通过后检查宏观叙事、审美方向、交付路线、证据链和风险，并在协作记录中签署通过/退回；退回时必须指向对应副主编，由副主编下发工作平台。
16. QA冻结：QA副主编只对带版本号冻结稿做逐页截图、交互验收、大屏验收和回归；冻结或未冻结结论必须写入协作记录。
17. 用户验收：提交最终交付物、截图包、测试记录、跨页一致性清单和条件附件；用户提出意见时，由总编分发给对应副主编，重新进入“副主编下发工作平台 → 工作平台修复 → 副主编复审 → 总编复核 → QA回归 → 用户复验”闭环。

## 执行规则

1. 每个关键事项必须有唯一A。
2. 如果同一人兼任多个角色，仍需按角色检查，不得跳过对应标准。
